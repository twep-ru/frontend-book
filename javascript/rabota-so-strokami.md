# Работа со строками

Понимание того, как работать с длиной строки, искать подстроки и изменять регистр, является фундаментом для любого фронтенд-разработчика, так как большая часть данных, получаемых от пользователя или сервера, представлена именно в текстовом виде.

#### Основы: длина строки и доступ к символам

Каждая строка обладает встроенным свойством `length`, которое возвращает общее количество символов в ней, включая пробелы и знаки препинания. Важно помнить, что `length` — это именно свойство, а не метод, поэтому оно не требует вызова с круглыми скобками.

Для получения конкретного символа из строки чаще всего используется синтаксис квадратных скобок `[]`, где указывается индекс нужного символа. В JavaScript индексация начинается с нуля, то есть первый символ находится под индексом `0`. Если вы попытаетесь обратиться к индексу, которого не существует, вы получите `undefined`.

Более современный и гибкий способ — метод `.at()`. В отличие от квадратных скобок, он позволяет использовать отрицательные индексы. Например, `.at(-1)` вернет последний символ строки, что избавляет от необходимости писать громоздкие конструкции вроде `str[str.length - 1]`.

```javascript
const userName = "Александр";
console.log(userName.length); // 9
console.log(userName[0]);     // "А"
console.log(userName.at(-1)); // "р"
```

#### Изменение регистра и очистка строки

Часто при сравнении строк или валидации форм необходимо привести текст к единому формату. Для этого используются методы `.toLowerCase()` (перевод в нижний регистр) и `.toUpperCase()` (перевод в верхний регистр).

Еще одной важной операцией является удаление лишних пробелов по краям строки, которые могут появиться при вводе данных пользователем. Метод `.trim()` полностью удаляет пробельные символы в начале и в конце строки, не затрагивая пробелы между словами. Также существуют точечные методы `.trimStart()` и `.trimEnd()` для очистки только одной стороны.

#### Поиск и проверка вхождения подстрок

Для проверки наличия определенного текста внутри строки в современном JavaScript используется метод `.includes()`. Он возвращает логическое значение `true` или `false`. Если же вам нужно узнать точную позицию (индекс) начала искомого текста, стоит использовать `.indexOf()`. Если подстрока не найдена, `.indexOf()` вернет `-1`.

Дополнительно существуют методы `.startsWith()` и `.endsWith()`, которые позволяют проверить, начинается или заканчивается ли строка на определенную последовательность символов. Всем этим методам можно передать второй аргумент — индекс, с которого следует начинать поиск.

```javascript
const message = "Привет, фронтенд!";
console.log(message.includes("фронтенд")); // true
console.log(message.startsWith("Привет")); // true
```

#### Извлечение частей строки и замена данных

Если необходимо вырезать часть текста, лучше всего использовать метод `.slice()`. Он принимает два аргумента: индекс начала и индекс конца (не включая его). Метод `.slice()` удобен тем, что поддерживает отрицательные индексы для отсчета с конца строки. Существует также метод `.substring()`, но он менее функционален, так как не умеет работать с отрицательными значениями.

Для замены текста используются методы `.replace()` (заменяет первое найденное вхождение) и `.replaceAll()` (заменяет все совпадения). В качестве искомого значения можно передавать не только обычную строку, но и регулярное выражение для более сложных замен.

#### Разбиение строки на массив

Метод `.split()` — незаменимый инструмент, когда нужно превратить строку в массив. Он принимает разделитель (например, пробел, запятую или пустую строку) и делит текст на части в местах совпадения с этим разделителем. Это крайне полезно при обработке списков или анализе сложных текстовых структур.

```javascript
const tags = "js, react, css";
const tagsArray = tags.split(", "); 
// ["js", "react", "css"]
```

Важнейшее правило, которое нужно усвоить: строки в JavaScript иммутабельны (неизменяемы). Ни один из перечисленных методов не меняет оригинальную строку, а всегда возвращает новую. Чтобы сохранить изменения, результат работы метода нужно записать в новую переменную или перезаписать текущую.

#### Итог

Мы изучили основные инструменты для работы с текстом: от получения длины строки и доступа к отдельным символам до методов поиска, обрезки и замены данных. Понимание того, что строки нельзя изменить напрямую и что методы всегда возвращают новый результат, является ключевым для предотвращения ошибок в логике приложений.

#### Задача на закрепление

Напишите программу для базовой проверки и форматирования почтового адреса (email), введенного пользователем.

1. Создайте переменную `userEmail`, содержащую строку с почтой (например, `" MyEmail@Gmail.Com "`).
2. Очистите строку от лишних пробелов по краям.
3. Приведите всю строку к нижнему регистру.
4. Проверьте, содержит ли строка символ `@` и заканчивается ли она на `.com`. Выведите результат проверки в консоль.
5. Замените в очищенной строке домен `gmail.com` на `yandex.ru` и выведите финальный результат.

**Критерии успешного выполнения:**

* В консоли выведен результат проверки (true/false).
* Финальная строка выведена без пробелов и в нижнем регистре.
* Замена домена произведена корректно.
