# Навигация по элементам дерева

Для фронтенд-разработчика понимание того, как перемещаться по этому дереву и как правильно подключать скрипты, чтобы они имели доступ к элементам, является фундаментальным навыком. Без этих знаний невозможно создать интерактивный интерфейс, так как скрипт может просто «не увидеть» нужную кнопку или форму, если он запустится слишком рано.

### Подключение скриптов: async и defer

По умолчанию, когда браузер встречает тег `<script>` в HTML-файле, он приостанавливает разбор (парсинг) страницы, загружает скрипт и выполняет его. Если скрипт находится в блоке `<head>`, попытка обратиться к элементам внутри `<body>` вернет `null`, так как браузер их еще не создал. Простейшее решение — перенести скрипт в самый конец `<body>`, но современный стандарт предлагает более элегантные способы с помощью атрибутов.

Атрибут defer заставляет браузер загружать скрипт в фоновом режиме, не прерывая чтение HTML. При этом выполнение скрипта произойдет строго тогда, когда всё дерево DOM будет полностью готово. Это гарантирует, что любая часть страницы будет доступна вашему коду.

Атрибут async также загружает скрипт в фоне, но выполняет его сразу после загрузки, не дожидаясь готовности всей страницы. Это может быть опасно, если скрипт должен работать с элементами DOM, так как момент его запуска непредсказуем. Обычно `async` используют для независимых скриптов, например, систем аналитики или метрик.

### Точки входа в документ

Весь DOM доступен через глобальный объект `document`. Чтобы получить быстрый доступ к самым верхним узлам дерева, существуют встроенные свойства:

* `document.documentElement` — ссылка на тег `<html>`.
* `document.head` — доступ к содержимому тега `<head>`.
* `document.body` — доступ к тегу `<body>`.

Важно помнить, что если вы обратитесь к `document.body` внутри скрипта в `<head>` без атрибута `defer`, вы получите `null`, так как на тот момент браузер еще не дошел до обработки тела документа.

### Навигация по элементам дерева

Когда у вас уже есть ссылка на какой-то элемент, вы можете «путешествовать» по дереву к его соседям, родителям или детям. Существует набор свойств, которые работают исключительно с элементами (тегами), игнорируя текстовые узлы и комментарии.

Свойство `parentElement` позволяет подняться на один уровень вверх к родителю. Для перемещения по одному уровню (между соседями) используются `previousElementSibling` (предыдущий сосед) и `nextElementSibling` (следующий сосед). Если соседа нет, свойство вернет `null`.

Для работы с вложенными структурами используется свойство `children`. Оно возвращает HTML-коллекцию, которая содержит всех прямых потомков элемента. Также удобно использовать свойства `firstElementChild` и `lastElementChild` для быстрого доступа к первому и последнему ребенку соответственно.

```javascript
const list = document.querySelector('ul');

const firstItem = list.firstElementChild; // Первая <li>
const nextItem = firstItem.nextElementSibling; // Вторая <li>
const parent = list.parentElement; // Родитель списка (например, <div> или <body>)
```

Стоит отметить, что `children` возвращает не массив, а «живую» коллекцию. Она автоматически обновляется при добавлении или удалении элементов в DOM. Если вам нужны методы массивов (например, `forEach`), коллекцию нужно преобразовать с помощью `Array.from(collection)` или спред-оператора `[...collection]`.

### Разница между узлами и элементами

В DOM-дереве всё является узлом (node): и сами теги (элементы), и текст внутри них, и даже символы переноса строки или пробелы. Свойства навигации, которые мы рассмотрели выше (с приставкой `Element`), сфокусированы только на тегах. Однако существуют и более общие свойства для работы со всеми типами узлов.

Например, `childNodes` вернет коллекцию абсолютно всех дочерних узлов, включая текстовые. Аналогично, `firstChild`, `lastChild`, `previousSibling` и `nextSibling` будут «видеть» переносы строк и пробелы в коде как отдельные объекты. В повседневной фронтенд-разработке чаще всего требуется манипулировать именно элементами, поэтому свойства с `ElementChild` или `ElementSibling` используются гораздо активнее.

### Итог

Мы изучили механизмы взаимодействия JavaScript с DOM-деревом, узнав, как атрибуты `defer` и `async` влияют на доступность элементов при загрузке страницы. Понимание разницы между навигацией по узлам и навигацией по элементам позволяет более точно выбирать цели для манипуляций. Владение такими свойствами, как `parentElement`, `children` и `nextElementSibling`, является основой для динамического изменения интерфейса и работы с данными на странице.

### Задача на закрепление

Представьте, что у вас есть следующая разметка:

```html
<div id="container">
  <h1>Заголовок</h1>
  <p>Первый абзац</p>
  <ul>
    <li>Пункт 1</li>
    <li>Пункт 2</li>
  </ul>
</div>
```

**Выполните следующие действия через консоль браузера или в отдельном скрипте:**

1. Получите ссылку на элемент `ul`, используя навигацию от заголовка `h1` (не используйте поиск по ID или классу).
2. Выведите в консоль количество дочерних элементов (тегов) внутри `div#container`.
3. Получите текст «Пункт 2», начав путь от элемента `ul`.
4. Проверьте, что является родителем для `document.documentElement`, используя свойства `parentElement` и `parentNode`. Объясните разницу в результате.
5. Преобразуйте коллекцию `children` элемента `ul` в настоящий массив и выведите его в консоль.

**Критерии успешного выполнения:**

* В первом пункте использована цепочка свойств (например, `nextElementSibling`).
* Код корректно различает дочерние элементы (`children`) и дочерние узлы.
* Продемонстрировано понимание разницы между `parentElement` и `parentNode` на корневом элементе.
* Использован метод `Array.from()` или `[...]` для преобразования коллекции.

Желаю успехов в практике! Если возникнут сложности с навигацией по "соседям", попробуйте отрисовать дерево элементов на бумаге — это отлично помогает визуализировать связи.
