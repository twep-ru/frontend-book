# Переменные

Они позволяют нам сохранять данные в памяти компьютера под понятными именами, чтобы мы могли обращаться к ним, изменять их и использовать в логике приложения. Без переменных было бы невозможно создать даже простейшую интерактивную форму или обработать ответ от сервера, так как нам просто негде было бы хранить промежуточные результаты вычислений.

### Сущность и объявление переменных

Переменная — это своего рода «контейнер» или именованная область памяти, предназначенная для хранения данных. В JavaScript для создания переменной используются ключевые слова `let`, `const` или устаревшее `var`. Процесс создания переменной называется объявлением. Сразу после объявления мы можем присвоить ей значение с помощью оператора присваивания `=`.

```javascript
let message; // Объявление
message = "Привет!"; // Присваивание значения
```

Для JavaScript характерна динамическая типизация. Это означает, что вам не нужно заранее указывать, какой тип данных (текст или число) будет храниться в переменной. Интерпретатор сам определит тип на основе присвоенного значения. Более того, одну и ту же переменную, объявленную через `let`, можно переопределять сколько угодно раз, записывая в неё новые данные.

### Различия между let, const и var

Выбор правильного ключевого слова для объявления переменной напрямую влияет на стабильность вашего кода. Современный стандарт разработки предписывает использовать `let` для переменных, значение которых будет меняться, и `const` для констант — значений, которые остаются неизменными на протяжении всей работы программы. Если вы попытаетесь переопределить значение переменной, созданной через `const`, JavaScript выдаст ошибку, что помогает избежать случайных багов.

Исторически в языке существовало только ключевое слово `var`. Его главная проблема заключается в специфическом поведении: переменные, объявленные через `var`, можно использовать еще до того, как они были фактически созданы в коде (так называемое «всплытие»). Это часто приводило к трудноотловимым ошибкам, поэтому в современном фронтенде использование `var` считается плохой практикой.

### Строгий режим (use strict)

JavaScript развивался десятилетиями, и многие ранние решения со временем стали считаться неудачными. Чтобы исправить ошибки прошлого, не ломая при этом старые сайты, был введен строгий режим — `"use strict"`. Эта специальная инструкция в начале файла заставляет интерпретатор работать по современным стандартам и строже относиться к синтаксическим неточностям.

Например, в обычном режиме JavaScript позволяет создать переменную, просто написав её имя без ключевого слова `let` или `const`. Это плохая практика, которая замусоривает глобальную область видимости. В строгом режиме такая попытка приведет к немедленной ошибке, заставляя разработчика писать чистый и предсказуемый код. Рекомендуется всегда включать строгий режим в начале своих скриптов.

### Правила именования переменных

Имя переменной должно быть максимально описательным. Прошли времена, когда переменные называли одной буквой `a` или `b`; сегодня во фронтенде принято использовать стиль camelCase (верблюжья нотация). В этом стиле первое слово пишется с маленькой буквы, а каждое последующее — с большой, например: `userName`, `totalPrice` или `isAccountActive`.

Существуют и технические ограничения:

* Имя может содержать буквы, цифры, символы `$` и `_`.
* Имя не может начинаться с цифры.
* Нельзя использовать зарезервированные слова (например, `if`, `class`, `function`), так как они уже заняты самим языком для выполнения команд.

### Итог

Мы изучили основы работы с данными через переменные, узнали о различиях между современными способами объявления `let`, `const` и устаревшим `var`. Понимание строгого режима и соблюдение правил именования в стиле camelCase помогают писать профессиональный код, который будет понятен не только компьютеру, но и другим разработчикам в команде. Эти базовые концепции открывают дверь к изучению типов данных и сложной логики приложений.

### Задача на закрепление

Представьте, что вы разрабатываете систему управления интернет-магазином. Вам необходимо объявить данные о товаре и пользователе.

1. Включите в скрипте строгий режим.
2. Объявите переменную для названия магазина, которая не будет меняться.
3. Объявите переменную для имени текущего посетителя, используя `let`.
4. Создайте переменную для цены товара и выведите в консоль сообщение, объединяющее имя пользователя и цену (например: "Александр, цена вашего заказа: 500").
5. Измените цену товара и снова выведите аналогичное сообщение в консоль.
6. Попробуйте изменить название магазина и посмотрите на ошибку в консоли.

Критерии успешного выполнения:

* Скрипт начинается с `"use strict"`.
* Имена переменных соответствуют стилю camelCase.
* Использованы правильные ключевые слова (`const` для неизменных данных, `let` для изменяемых).
* В консоли отображаются корректные сообщения после изменения цены.
