# Типы данных

Понимание того, как язык обрабатывает эти типы, как работает автоматическое приведение и как проверить текущий тип переменной, является фундаментом для написания предсказуемого и качественного кода.

### Примитивные типы данных и строки

Примитивные типы данных хранят в себе простые значения. К ним относятся строки (`string`), числа (`number`), большие числа (`bigint`), логические значения (`boolean`), а также специальные значения `null` и `undefined`.

Работа со строками в современном JavaScript часто задействует обратные кавычки (backticks). Этот синтаксис позволяет выполнять интерполяцию — встраивание выражений или переменных прямо внутрь строки с помощью конструкции `${}`. Кроме того, только обратные кавычки позволяют переносить текст на новую строку без использования специальных символов.

```javascript
const name = "Александр";
const message = `Привет, ${name}!
Добро пожаловать в мир JavaScript.`;
```

### Особенности чисел и BigInt

Числовой тип `number` охватывает как целые числа, так и числа с плавающей точкой. Важно учитывать существование специальных числовых значений: `Infinity` (бесконечность) и `NaN` (Not a Number). `NaN` обычно появляется в результате математически невозможных операций, например, при попытке разделить строку на число.

Стандартный тип `number` имеет ограничение по точности для очень больших чисел (безопасный диапазон — от $$ $-2^{53} + 1$ $$ до $$ $2^{53} - 1$ $$). Для работы с целыми числами произвольной длины используется тип `BigInt`, который создается путем добавления суффикса `n` к числу. Стоит помнить, что смешивать `BigInt` и обычные числа в одной операции нельзя без явного преобразования.

### Проверка типов с помощью typeof

Для определения типа данных используется оператор `typeof`. Он возвращает строку с названием типа. Однако у этого оператора есть историческая особенность: вызов `typeof null` возвращает `"object"`. Это признанная ошибка в языке, которую невозможно исправить из-за обратной совместимости, поэтому `null` следует воспринимать как намеренное отсутствие значения, а не как реальный объект.

```javascript
console.log(typeof 42);           // "number"
console.log(typeof "Текст");      // "string"
console.log(typeof true);         // "boolean"
console.log(typeof undefined);    // "undefined"
console.log(typeof null);         // "object" (баг языка)
```

### Преобразование типов данных

Преобразование может быть неявным (автоматическим) и явным. Неявное преобразование часто происходит при использовании оператора сложения со строкой: JavaScript превратит число в строку и выполнит склейку. Явное преобразование выполняется с помощью функций `String()`, `Number()` и `Boolean()`.

При преобразовании в логический тип (`boolean`) важно запомнить список "ложных" (falsy) значений, которые всегда превращаются в `false`: это число `0`, пустая строка `""`, `null`, `undefined` и `NaN`. Все остальные значения, включая отрицательные числа и строки с пробелами, преобразуются в `true`.

### Итог

Мы рассмотрели основные кирпичики JavaScript — его типы данных. Вы научились использовать интерполяцию строк для создания динамических сообщений, узнали о специфике работы с большими числами и научились безопасно проверять типы переменных. Понимание механизмов преобразования типов поможет вам избегать трудноловимых ошибок при выполнении арифметических операций и логических проверок в ваших будущих проектах.

### Задача на закрепление

Создайте скрипт для обработки данных пользователя.

1. Объявите переменные для хранения имени (`string`), возраста (`number`) и статуса подписки (`boolean`).
2. Используя интерполяцию строк, сформируйте приветственное сообщение, которое включает в себя имя и возраст.
3. Проверьте тип каждой переменной с помощью `typeof` и выведите результат в консоль.
4. Выполните явное преобразование: преобразуйте числовой возраст в строку, а строку `"100"` в число. Выведите их новые типы.
5. Создайте переменную со значением `null` и проверьте, что выдаст `typeof`.

**Критерии успешного выполнения:**

* В консоли отображается корректно сформированная строка с данными пользователя.
* Для всех типов данных (включая преобразованные) выведены правильные результаты `typeof`.
* Продемонстрировано понимание разницы между строковым числом и типом `number`.
