# Размеры и координаты DOM-элементов

Это необходимо для реализации таких задач, как «ленивая» загрузка изображений, создание бесконечных лент новостей, позиционирование всплывающих подсказок или создание плавной навигации по разделам лендинга. Работа с метриками окна позволяет разработчику точно контролировать, что видит пользователь в конкретный момент времени и как элементы взаимодействуют с границами экрана.

### Размеры окна браузера (вьюпорта)

Для получения ширины и высоты текущего окна (вьюпорта) наиболее надежным способом является обращение к корневому элементу документа через `document.documentElement`. Свойства `clientWidth` и `clientHeight` возвращают размеры области, доступной для контента, исключая полосы прокрутки.

Существует альтернативный способ получения размеров через глобальный объект `window` и его свойства `innerWidth` и `innerHeight`. Однако между ними есть существенная разница: свойства объекта `window` учитывают ширину полос прокрутки, если они есть. Для фронтенд-разработчика важнее знать именно «чистое» пространство, в котором можно располагать элементы, поэтому использование `document.documentElement.clientWidth` и `clientHeight` считается более правильным подходом при расчете координат.

```javascript
// Правильный способ получения размеров вьюпорта
const width = document.documentElement.clientWidth;
const height = document.documentElement.clientHeight;

console.log(`Ширина окна: ${width}, Высота окна: ${height}`);
```

### Определение размеров всей страницы

Когда необходимо узнать полную высоту или ширину страницы с учетом всей прокручиваемой области, используются свойства `scrollWidth` и `scrollHeight` элемента `document.documentElement`. Важно учитывать, что из-за особенностей рендеринга в разных браузерах значение `scrollHeight` иногда может вычисляться некорректно.

Для получения максимально точного значения полной высоты документа рекомендуется использовать проверку через `Math.max`, сравнивая несколько метрик (ClientHeight, ScrollHeight, OffsetHeight) как для `document.body`, так и для `document.documentElement`. Такой подход гарантирует, что вы получите наибольшее и наиболее корректное значение, вне зависимости от специфики конкретного браузера.

```javascript
const scrollHeight = Math.max(
  document.body.scrollHeight, document.documentElement.scrollHeight,
  document.body.offsetHeight, document.documentElement.offsetHeight,
  document.body.clientHeight, document.documentElement.clientHeight
);
```

### Текущая позиция прокрутки

Чтобы узнать, на каком расстоянии от верха или левого края страницы сейчас находится пользователь, используются свойства `window.scrollX` (для горизонтальной прокрутки) и `window.scrollY` (для вертикальной). В старых учебниках можно встретить свойства `pageXOffset` и `pageYOffset`, но в современной разработке они считаются устаревшими (deprecated), и их использование не рекомендуется. Эти координаты позволяют определять, дошел ли пользователь до определенного блока или пора ли показывать кнопку «Наверх».

### Методы программного управления скроллом

JavaScript предоставляет несколько способов для управления прокруткой страницы. Методы `scroll()` и `scrollTo()` выполняют одну и ту же задачу: они перемещают страницу к абсолютным координатам относительно начала документа. Например, вызов `window.scroll(0, 500)` переместит пользователя к отметке 500 пикселей от верха, независимо от того, где он находился до этого.

Метод `scrollBy()`, напротив, работает относительно текущей позиции. Если пользователь находится на отметке 200 пикселей и вызывается `window.scrollBy(0, 100)`, страница сдвинется еще на 100 пикселей вниз, оказавшись на отметке 300. Для всех этих методов можно передать объект с параметром `behavior: 'smooth'`, чтобы прокрутка была плавной, а не мгновенной.

```javascript
// Плавная прокрутка на 200 пикселей вниз от текущего места
window.scrollBy({
  top: 200,
  behavior: 'smooth'
});
```

### Метод scrollIntoView

Одним из самых мощных инструментов является метод `scrollIntoView()`, который вызывается непосредственно у DOM-элемента, до которого нужно прокрутить страницу. Это избавляет от необходимости вручную высчитывать координаты элемента.

Метод принимает объект настроек, где свойство `block` определяет, как именно элемент должен расположиться в окне после прокрутки:

* `start` — верхняя граница элемента совпадет с верхней границей окна (по умолчанию).
* `end` — нижняя граница элемента совпадет с нижней границей окна.
* `center` — элемент окажется ровно по центру вьюпорта.
* `nearest` — браузер выберет ближайшую границу для минимального движения.

### Итог

Мы изучили ключевые метрики браузерного окна и способы управления поведением страницы. Теперь вы знаете разницу между размерами вьюпорта и полной высотой страницы, умеете получать текущие координаты прокрутки и использовать методы для перемещения пользователя по контенту. Главное — помнить о различиях между абсолютным позиционированием через `scroll` и относительным через `scrollBy`, а также использовать `scrollIntoView` для удобной навигации к конкретным элементам интерфейса.

   * Текущую позицию прокрутки по вертикали.

* Для прокрутки используется метод `scrollIntoView` с правильными параметрами.
* Метрики окна определяются через `document.documentElement`.
* Все данные корректно отображаются в консоли при вызове функций.
