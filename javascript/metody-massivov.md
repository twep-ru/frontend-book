# Массивы, часть 2

Будь то список товаров в интернет-магазине, сообщения в чате или настройки пользователя, все это представляется в виде массивов, которые необходимо фильтровать, преобразовывать и анализировать. Понимание методов перебора массивов в JavaScript позволяет писать чистый, декларативный код, который легче читать и поддерживать, заменяя громоздкие циклы `for` элегантными цепочками методов.

### Основы и деструктуризация

Прежде чем переходить к методам перебора, важно вспомнить о деструктурирующем присваивании, которое часто используется вместе с ними. Деструктуризация позволяет извлекать значения из массивов и присваивать их переменным в одну строку. Например, если у нас есть массив с координатами `[10, 20]`, мы можем легко получить `x` и `y`, не обращаясь к элементам по индексам. Это делает код более выразительным, особенно когда мы работаем с результатами функций или сложными структурами данных.

### Базовый перебор: forEach и map

Самый простой способ пройтись по всем элементам — метод `forEach`. Он выполняет указанную функцию один раз для каждого элемента массива. Однако стоит помнить, что `forEach` ничего не возвращает (его результат — `undefined`). Он идеален для создания побочных эффектов, например, вывода данных в консоль или обновления внешнего состояния.

```javascript
const fruits = ['apple', 'banana', 'orange'];
fruits.forEach((item) => {
  console.log(`Фрукт: ${item}`);
});
```

Если же ваша цель — создать новый массив на основе существующего, следует использовать `map`. Этот метод трансформирует каждый элемент и возвращает новый массив той же длины. Это ключевой инструмент в React для рендеринга списков компонентов.

```javascript
const numbers = [1, 2, 3];
const squares = numbers.map(num => num * num); // [1, 4, 9]
```

### Фильтрация и поиск: filter, find, indexOf

Когда нужно отобрать только те элементы, которые соответствуют определенному условию, на помощь приходит `filter`. Он возвращает новый массив, содержащий только те элементы, для которых функция-колбэк вернула `true`. Если же нам нужен всего один, первый попавшийся элемент, используется `find`.

Для поиска индекса элемента или проверки его наличия применяются методы `indexOf` (возвращает индекс или -1) и `includes` (возвращает `true/false`). `includes` особенно удобен для простых проверок на наличие примитивных значений в массиве.

```javascript
const users = [
  { name: 'Ivan', age: 17 },
  { name: 'Anna', age: 25 },
  { name: 'Oleg', age: 30 }
];

const adults = users.filter(user => user.age >= 18);
```

### Проверка условий: some и every

Иногда нам не нужно извлекать данные, а достаточно лишь получить логический ответ на вопрос о состоянии всего массива. Метод `some` вернет `true`, если хотя бы один элемент соответствует условию. Напротив, `every` требует, чтобы условию соответствовали абсолютно все элементы массива. Эти методы работают эффективно: они прекращают перебор, как только результат становится очевиден.

```javascript
const tasks = [
  { id: 1, completed: true },
  { id: 2, completed: false }
];

const hasIncomplete = tasks.some(task => !task.completed); // true
const allCompleted = tasks.every(task => task.completed); // false
```

### Сортировка: метод sort

Метод `sort` изменяет исходный массив, выстраивая его элементы в определенном порядке. По умолчанию он сортирует элементы как строки, что может привести к странным результатам с числами (например, "10" окажется раньше "2"). Чтобы отсортировать числа или объекты правильно, в `sort` нужно передать функцию сравнения.

```javascript
const scores = [40, 100, 1, 5];
scores.sort((a, b) => a - b); // [1, 5, 40, 100]
```

#### Мощный инструмент агрегации: reduce

Метод `reduce` является одним из самых гибких и в то же время сложных для освоения. Он используется для вычисления единого значения на основе всего массива — будь то сумма чисел, объединение строк или создание объекта из массива. Он принимает аккумулятор (накопитель) и текущий элемент, постепенно "сворачивая" массив.

```javascript
const prices = [100, 200, 300];
const total = prices.reduce((acc, current) => acc + current, 0); // 600
```

### Итог

Изучение методов массивов радикально меняет подход к написанию кода, делая его более предсказуемым и лаконичным. Мы рассмотрели, как трансформировать данные с помощью `map`, отбирать нужное через `filter`, проводить логические проверки с `some/every` и агрегировать значения через `reduce`. Важно помнить, что большинство этих методов не изменяют исходный массив, а возвращают новый, что соответствует принципам иммутабельности в современном фронтенде. Комбинируя эти инструменты, вы сможете эффективно решать практически любые задачи по обработке данных на стороне клиента.

### Задача на закрепление

Представьте, что вы разрабатываете систему управления онлайн-магазином. У вас есть массив объектов `products`:

```javascript
const products = [
  { name: 'Laptop', price: 1200, category: 'Electronics' },
  { name: 'Phone', price: 800, category: 'Electronics' },
  { name: 'Shirt', price: 40, category: 'Clothing' },
  { name: 'Shoes', price: 80, category: 'Clothing' },
  { name: 'Watch', price: 200, category: 'Electronics' }
];
```

**Ваша задача:**

1. Создайте новый массив, содержащий только товары из категории `Electronics`.
2. Из этого списка отфильтрованных товаров сформируйте массив строк, где каждая строка имеет вид: "Товар: \[название], Цена: $\[цена]".
3. Используя метод `reduce`, вычислите общую стоимость всех товаров в категории `Electronics`.

**Критерии успешного выполнения:**

* Оригинальный массив `products` не должен быть изменен.
* Для каждого этапа использован соответствующий метод массива (`filter`, `map`, `reduce`).
* Код выводит в консоль итоговый список строк и общую сумму.
