# Взаимодействие с пользователем

Для этого в JavaScript предусмотрены встроенные механизмы, позволяющие выводить уведомления, запрашивать ввод данных или получать подтверждение действий. Кроме того, когда логика приложения требует сравнения одной переменной с множеством вариантов, на помощь приходит конструкция `switch`, которая делает код более структурированным и читаемым по сравнению с длинными цепочками условий `if...else`.

### Модальные окна: Alert, Prompt и Confirm

Браузер предоставляет три основные функции для прямого диалога с пользователем. Все они создают модальные окна, которые временно приостанавливают выполнение остального скрипта на странице до тех пор, пока пользователь не закроет их.

Самая простая функция — `alert()`. Она используется для вывода информационного сообщения. В круглых скобках передается значение, которое автоматически приводится к строке и отображается в окне с единственной кнопкой «ОК». Это полезно для уведомлений о завершении процесса или ошибках.

Для получения информации от пользователя применяется `prompt()`. Эта функция принимает два аргумента: заголовок (вопрос) и необязательное значение по умолчанию, которое будет подставлено в поле ввода. Если пользователь вводит текст и нажимает «ОК», функция возвращает этот текст в виде строки. Важно помнить: если нажать «Отмена» или закрыть окно клавишей Escape, функция вернет специальное значение `null`. Так как `prompt` всегда работает со строками, при получении чисел их необходимо преобразовывать с помощью унарного плюса или функции `Number()`, иначе математические операции могут дать неверный результат.

Функция `confirm()` служит для получения простого согласия или отказа. Она выводит окно с вопросом и двумя кнопками: «ОК» и «Отмена». Результат работы этой функции всегда имеет булев тип: `true`, если пользователь подтвердил действие, и `false`, если отказался.

### Конструкция Switch: логика множественного выбора

Когда необходимо сравнить значение переменной с несколькими конкретными вариантами, использование `switch` становится наиболее изящным решением. В скобках после ключевого слова `switch` указывается выражение, которое будет последовательно сравниваться со значениями, описанными в блоках `case`.

```javascript
const userRole = "admin";

switch (userRole) {
  case "admin":
    console.log("Доступ ко всем настройкам разрешен");
    break;
  case "editor":
    console.log("Доступ только к редактированию контента");
    break;
  case "guest":
    console.log("Доступ только для чтения");
    break;
  default:
    console.log("Роль не определена");
}
```

Одной из самых важных деталей в этой конструкции является оператор `break`. Он прерывает выполнение `switch` после того, как совпадение найдено и соответствующий код выполнен. Если забыть написать `break`, JavaScript продолжит автоматически выполнять все последующие блоки `case` ниже по списку, даже если условия в них ложны. Блок `default` является аналогом `else` и срабатывает, если ни одно из значений `case` не совпало. Также следует учитывать, что `switch` всегда использует строгое сравнение (аналог оператора `===`), поэтому типы данных должны совпадать точно: число `0` не будет равно строке `"0"`.

В случаях, когда нужно проверить диапазоны значений (например, возраст в разных промежутках), применяется подход `switch (true)`. В этом варианте после каждого `case` записывается полноценное логическое выражение, и выполнится тот блок, условие которого первым окажется истинным.

### Итог

Мы изучили инструменты взаимодействия с пользователем через браузерные модальные окна, которые позволяют гибко управлять вводом и подтверждением данных, хотя и блокируют основной поток выполнения кода. Параллельно мы освоили оператор `switch case` как эффективный способ организации множественных проверок, подчеркнув важность строгого сравнения типов и обязательного использования оператора `break` для предотвращения логических ошибок.

### Задача на закрепление

Создайте программу «Мини-калькулятор», которая взаимодействует с пользователем и выполняет базовые расчеты.

1. Используя `confirm`, спросите пользователя: «Хотите выполнить математическую операцию?». Если пользователь отказывается, выведите в консоль сообщение «Работа завершена» и не продолжайте.
2. С помощью двух вызовов `prompt` запросите у пользователя два числа. Обязательно приведите результат к числовому типу.
3. С помощью третьего `prompt` запросите знак операции: `+`, `-`, `*` или `/`.
4. Используя конструкцию `switch`, выполните соответствующее действие с числами.
5. Результат выведите в модальном окне через `alert`.
6. Добавьте в `switch` блок `default`, который выведет ошибку «Неизвестная операция», если пользователь ввел некорректный символ.

**Критерии успешного выполнения:**

* Программа корректно складывает числа (например, `2 + 2 = 4`, а не `"22"`).
* Если пользователь нажимает «Отмена» в самом начале, расчет не запускается.
* Для выбора операции используется `switch` с обязательными `break`.
* Отработана ситуация ввода неверного знака операции через `default`.
