# DOM-манипуляции

Умение динамически изменять страницу, создавать новые элементы и перемещать существующие — это то, что превращает статичную HTML-разметку в живое, интерактивное приложение. Мы изучим, как читать и перезаписывать содержимое узлов, как эффективно добавлять элементы в документ и в чем разница между различными методами вставки.

### Работа с содержимым элементов: textContent и innerHTML

Первое, с чем сталкивается разработчик — это необходимость получить или изменить текст внутри элемента. Для этого чаще всего используется свойство `textContent`. Оно позволяет прочитать весь текст внутри узла, включая текст из всех вложенных дочерних элементов, объединяя их в одну строку. Если присвоить этому свойству новое значение, старое содержимое (включая все вложенные теги) будет полностью удалено и заменено обычным текстом. Важно помнить, что `textContent` воспринимает любую переданную строку именно как текст, поэтому попытка вставить туда HTML-разметку приведет к тому, что пользователь увидит теги прямо на экране.

Если же ваша задача — внедрить полноценную HTML-структуру, на помощь приходит `innerHTML`. Это свойство позволяет работать с внутренней разметкой элемента. При чтении оно возвращает строку со всеми вложенными тегами, а при записи интерпретирует строку как HTML, создавая соответствующие элементы в DOM. Однако стоит быть осторожным: использование `innerHTML` для добавления элементов через оператор `+=` является ресурсозатратным. Браузеру приходится полностью пересобирать всё содержимое узла, что может замедлить работу приложения при большом объеме данных.

### Создание и вставка новых элементов

Более производительный и современный способ работы с DOM — создание отдельных узлов с помощью метода `document.createElement()`. Этот метод создает элемент в памяти, но не добавляет его на страницу автоматически. После создания вы можете настроить объект: добавить ему классы через `classList.add()`, установить атрибуты или наполнить текстом через тот же `textContent`.

Для того чтобы элемент появился в документе, используются методы вставки. Существует пять основных способов размещения узла относительно выбранного родителя или соседа:

* `append()` — добавляет элемент в самый конец выбранного родительского узла.
* `prepend()` — вставляет элемент в начало родителя, перед всеми остальными дочерними узлами.
* `before()` — размещает новый узел непосредственно перед выбранным элементом.
* `after()` — вставляет узел сразу после выбранного элемента.
* `replaceWith()` — полностью заменяет текущий элемент новым узлом.

Особенность этих методов заключается в том, что они могут принимать сразу несколько аргументов (узлов или строк текста), которые будут вставлены по порядку. Если вы попытаетесь вставить один и тот же созданный элемент в разные места страницы несколько раз, он не продублируется, а просто "переедет" на новое место, так как в JavaScript переменная хранит ссылку на конкретный уникальный объект в памяти.

### Точечная вставка: insertAdjacentHTML

Иногда возникает задача вставить готовую строку с разметкой в конкретное место, не перезаписывая всё содержимое родителя. Для этого идеально подходит метод `insertAdjacentHTML()`. Он принимает два аргумента: позицию и саму строку с HTML.

**Метод поддерживает четыре позиции вставки:**

1. `'beforebegin'` — перед самим элементом (аналог `before`).
2. `'afterbegin'` — внутри элемента, перед первым дочерним узлом (аналог `prepend`).
3. `'beforeend'` — внутри элемента, после последнего дочернего узла (аналог `append`).
4. `'afterend'` — после самого элемента (аналог `after`).

Аналогично работает метод `insertAdjacentElement()`, но он предназначен для вставки уже созданных объектов-узлов, а не строк.

### Удаление, клонирование и перемещение

Если элемент больше не нужен, его можно легко удалить из DOM-дерева, вызвав метод `remove()` непосредственно у этого узла.

Для случаев, когда нужно создать точную копию существующего элемента, используется метод `cloneNode()`. По умолчанию он выполняет "поверхностное" клонирование — создает копию только самого тега без его содержимого. Чтобы скопировать элемент вместе со всеми вложенными узлами и текстом, необходимо передать в метод аргумент `true` (глубокое клонирование).

Интересно, что для перемещения элементов не нужны специальные методы. Поскольку методы вставки (`append`, `after` и т.д.) работают со ссылками на объекты, достаточно взять уже существующий на странице элемент и вызвать метод вставки у другой цели. Браузер поймет, что узел нужно не копировать, а перенести из старого места в новое.

### Итог

Манипуляции с DOM строятся на понимании разницы между текстовым и HTML-содержимым, а также на умении выбирать правильный метод вставки. Мы изучили, как создавать элементы через `createElement`, настраивать их и размещать на странице с помощью `append`, `prepend` и других современных методов. Понимание работы ссылочного типа данных при перемещении узлов и разницы между глубоким и поверхностным клонированием позволяет эффективно управлять структурой документа, делая интерфейсы быстрыми и динамичными.

### Задача на закрепление

Создайте динамический список дел (ToDo), используя методы манипуляции DOM.

1. Создайте в HTML пустой контейнер `div` с классом `list-container`.
2. Напишите функцию `createTask(text)`, которая создает новый элемент `div` с классом `task-item`.
3. Внутри `task-item` должен быть текст (переданный в функцию) и кнопка "Удалить".
4. Используйте метод `prepend`, чтобы каждая новая задача добавлялась в начало списка.
5. Реализуйте функционал удаления: при клике на кнопку "Удалить" соответствующий элемент `task-item` должен исчезать из DOM.
6. Добавьте кнопку "Клонировать список", которая будет создавать глубокую копию всего контейнера `list-container` и вставлять её после оригинала.

**Критерии успешного выполнения:**

* Новые задачи корректно создаются через `createElement` и появляются в начале списка.
* Кнопка удаления вызывает метод `remove()` для конкретного узла задачи.
* Клонирование создает полную копию списка со всеми вложенными элементами.
* Код не использует `innerHTML` для добавления задач (используйте `createElement` и `append`/`prepend`).
