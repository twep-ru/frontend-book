# События фокуса и потери фокуса

Понимание того, как интерактивные элементы получают и теряют фокус, критически важно для фронтенд-разработчика. Это позволяет создавать доступные интерфейсы, грамотно валидировать формы и управлять навигацией с помощью клавиатуры, что делает пользовательский опыт более предсказуемым и удобным.

### События focus и blur

Событие `focus` возникает, когда интерактивный элемент, например поле ввода или кнопка, получает состояние фокуса. Это может произойти при клике мышью, использовании клавиши Tab для навигации или при программном вызове. Состояние фокуса визуально сопровождается подсветкой элемента в браузере и появлением каретки (мигающей вертикальной черты) в текстовых полях. Перебор элементов в обратную сторону осуществляется комбинацией Shift + Tab.

Событие `blur` является противоположностью `focus` и срабатывает в момент, когда элемент теряет активное состояние. При переходе фокуса с одного поля ввода на другое, первое поле генерирует событие `blur`, а следующее за ним — `focus`.

```javascript
const inputElement = document.querySelector('input');

inputElement.addEventListener('focus', () => {
  console.log('Элемент получил фокус');
});

inputElement.addEventListener('blur', () => {
  console.log('Элемент потерял фокус');
});
```

### Отличие blur от change

Начинающие разработчики часто путают событие `blur` с событием `change`, так как `change` также возникает при потере фокуса элементом. Ключевое отличие заключается в том, что `change` срабатывает только в том случае, если значение элемента изменилось с момента получения им фокуса. Если пользователь установил курсор в поле ввода, ввел текст, затем сразу же удалил его и убрал фокус (оставив значение идентичным изначальному), событие `change` не возникнет. Событие `blur` отработает в любом случае, независимо от того, менялись ли данные. Это делает `blur` более надежным выбором для задач, где нужно строго реагировать на уход пользователя из поля ввода.

### Программное управление фокусом

Разработчик может управлять фокусом программно с помощью специальных методов `focus()` и `blur()`, доступных у DOM-элементов. Вызов метода `focus()` заставляет элемент немедленно стать активным, а `blur()` — принудительно снимает фокус.

```javascript
const loginInput = document.querySelector('#login');

// Устанавливаем фокус программно
loginInput.focus();

// Убираем фокус через 2 секунды
setTimeout(() => {
  loginInput.blur();
}, 2000);
```

Несмотря на наличие метода `focus()`, для установки фокуса на элемент при первичной загрузке страницы лучше использовать HTML-атрибут `autofocus`. Это более правильный подход, однако в таком случае перехватить автоматически возникшее первичное событие `focus` через JavaScript не получится.

### Всплытие событий: focusin и focusout

Важной особенностью событий `focus` и `blur` является то, что они не всплывают по DOM-дереву от целевого элемента вверх. Это означает, что вы не можете повесить один обработчик на `document` или родительский контейнер, чтобы отлавливать получение фокуса его дочерними элементами.

Для решения этой архитектурной проблемы существуют альтернативные события: `focusin` и `focusout`. Они полностью аналогичны `focus` и `blur`, но поддерживают всплытие. Используя их, можно перехватывать изменения фокуса на уровне родительских элементов и использовать паттерн делегирования, проверяя целевой элемент через `event.target.matches()`.

```javascript
document.addEventListener('focusin', (event) => {
  if (event.target.matches('.form-input')) {
    console.log('Фокус получен элементом формы');
  }
});
```

### Фокусировка неинтерактивных элементов

По умолчанию состояние фокуса бывает только на интерактивных элементах. Если необходимо сделать фокусируемым неинтерактивный тег (например, `div` при создании кастомных селектов), ему нужно добавить атрибут `tabindex` со значением в виде числа.

Значение `0` позволяет элементу получать фокус, при этом браузер самостоятельно определит его порядок в списке навигации на основе положения в разметке. Отрицательное значение (чаще всего `-1`) делает элемент доступным для программного фокуса, но исключает его из последовательности навигации по клавише Tab. Положительные значения задают строгий порядковый номер навигации, но использование таких значений часто считается антипаттерном.

### Определение текущего элемента в фокусе

Иногда при отладке интерфейса, особенно при работе со скрытыми модальными окнами, бывает сложно визуально определить, куда ушел фокус. Для этих целей существует полезное свойство `document.activeElement`, которое указывает на DOM-элемент, находящийся в состоянии фокуса в данный момент. Эту фишку удобно использовать вместе с инструментом Live Expression (иконка глаза) во вкладке Console в DevTools, чтобы отслеживать перемещение фокуса по странице в режиме реального времени.

### Задача на закрепление

Создайте простую HTML-форму с двумя полями ввода (логин и пароль) и отдельным текстовым блоком для вывода системных сообщений. Напишите скрипт, который будет управлять взаимодействием с этими полями.

1. Добавьте к обоим полям ввода обработчики событий получения и потери фокуса. При получении фокуса граница активного поля должна становиться синей, а при потере — возвращаться к стандартному цвету.
2. Реализуйте делегирование событий: используйте всплывающие события фокуса на уровне всей формы или документа, чтобы при фокусе на любом дочернем `input` в текстовом блоке выводился ID или класс текущего активного поля. Используйте для этого свойство глобального объекта документа, хранящее активный элемент.
3. Добавьте на страницу обычный текстовый `div` и сделайте его доступным для получения фокуса с клавиатуры в общем потоке навигации по клавише Tab, не меняя сам HTML-тег на кнопку или ссылку.

**Критерии успешного выполнения:**

* При перемещении по полям ввода с помощью клавиши Tab границы элементов корректно подсвечиваются и сбрасываются.
* Обработчик события для вывода информации висит только на родительском элементе, но успешно перехватывает изменения фокуса.
* Текстовый блок динамически обновляется, отображая данные о текущем элементе в фокусе.
* Неинтерактивный `div` успешно получает фокус при последовательной навигации с клавиатуры.
