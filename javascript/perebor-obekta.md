# Перебор объекта

Понимание того, как превращать объекты в массивы и обратно, а также использование специализированных коллекций, таких как Map и Set, позволяет фронтенд-разработчику писать более чистый, быстрый и предсказуемый код при обработке сложных данных от API или управлении состоянием интерфейса.

### Преобразование объектов в массивы и обратно

Хотя объекты не являются итерируемыми сущностями в том же смысле, что и массивы, глобальный объект `Object` предоставляет три мощных метода, которые позволяют получить содержимое объекта в виде массива для последующей обработки. Метод `Object.keys(user)` возвращает массив строк, содержащий все ключи объекта. Если же нам нужны только данные, метод `Object.values(user)` сформирует массив из значений свойств.

Наиболее гибким является метод `Object.entries(user)`, который трансформирует объект в массив массивов, где каждый вложенный элемент — это пара `[ключ, значение]`. Это открывает доступ ко всем методам массивов, таким как `map`, `filter` или `forEach`.

```javascript
const user = { name: "Александр", age: 28, city: "Москва" };

// Получаем массив пар и перебираем его с деструктуризацией
Object.entries(user).forEach(([key, value]) => {
  console.log(`${key}: ${value}`);
});
```

Когда обработка данных завершена, и нам нужно вернуть структуру к виду обычного объекта, используется метод `Object.fromEntries()`. Он выполняет обратную операцию, принимая массив пар `[ключ, значение]` и создавая на их основе новый объект.

### Коллекция Map: за пределами строковых ключей

Стандартные объекты имеют существенное ограничение: их ключи всегда являются строками или символами. Если попытаться использовать число или другой объект в качестве ключа, JavaScript автоматически приведет его к строке, что может вызвать конфликты данных. Коллекция `Map` решает эту проблему, позволяя использовать в качестве ключей абсолютно любые типы данных.

Для работы с `Map` используются специальные методы: `set(key, value)` для добавления данных, `get(key)` для их получения и `has(key)` для проверки наличия ключа. В отличие от объектов, `Map` сохраняет порядок добавления элементов и имеет свойство `size`, которое мгновенно возвращает количество записей.

```javascript
const data = new Map();

data.set(1, "Числовой ключ");
data.set("1", "Строковый ключ");

console.log(data.size); // 2 (разные типы ключей не конфликтуют)
```

Перебор `Map` осуществляется с помощью цикла `for...of`, который на каждой итерации возвращает массив из ключа и значения. Также `Map` легко конвертируется в объект и обратно через упоминавшиеся ранее методы `Object.entries` и `Object.fromEntries`.

### Коллекция Set: работа с уникальными значениями

Часто в разработке возникает задача оставить в списке только уникальные элементы, например, при обработке тегов или идентификаторов. Коллекция `Set` — это особый вид множества, где каждое значение может встречаться только один раз. Если попытаться добавить в `Set` уже существующее значение через метод `add()`, коллекция просто проигнорирует это действие.

Одним из самых популярных способов применения `Set` является быстрая очистка массива от дубликатов. Для этого массив передается в конструктор `new Set(array)`, а затем результат превращается обратно в массив с помощью оператора расширения или `Array.from()`.

```javascript
const numbers = [1, 2, 2, 3, 1, 4];
const uniqueNumbers = [...new Set(numbers)]; // [1, 2, 3, 4]
```

Как и `Map`, коллекция `Set` поддерживает методы `delete()`, `has()`, `clear()` и свойство `size`. Хотя у `Set` есть методы `keys()` и `values()`, они возвращают одно и то же содержимое для обеспечения совместимости интерфейсов с другими коллекциями.

### Итог

Мы изучили методы `Object.keys`, `values` и `entries`, которые стирают грань между объектами и массивами, позволяя трансформировать данные в обе стороны. Рассмотренные коллекции `Map` и `Set` расширяют возможности стандартного языка, предлагая гибкость в выборе типов ключей и автоматическую фильтрацию уникальных значений, что является критически важным инструментом в арсенале современного разработчика.

### Задача на закрепление

Представьте, что вы разрабатываете систему управления библиотекой.

1. Создайте массив `rawBooks`, содержащий названия книг, где некоторые названия повторяются.
2. Используя `Set`, создайте массив `uniqueBooks`, в котором будут только уникальные названия.
3. Создайте `Map` с названием `bookRatings`.
4. Добавьте в `bookRatings` три книги из вашего уникального списка в качестве ключей, а их рейтинги (числа от 1 до 10) — в качестве значений.
5. Используя цикл `for...of`, выведите в консоль сообщение для каждой книги: «Книга \[название] имеет рейтинг \[значение]».
6. Преобразуйте ваш `Map` обратно в обычный объект с помощью `Object.fromEntries`.

**Критерии успешного выполнения:**

* В итоговом массиве `uniqueBooks` отсутствуют дубликаты.
* В консоль корректно выводятся данные из `Map` через цикл.
* Финальный объект содержит те же пары ключ-значение, что и `Map`.
