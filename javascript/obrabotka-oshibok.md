# Обработка ошибок

Даже если ваш код написан идеально с точки зрения синтаксиса, внешние факторы, такие как некорректный ответ от сервера или неожиданные действия пользователя, могут привести к сбою. Для фронтенд-разработчика критически важно уметь предвидеть такие ситуации и обрабатывать их так, чтобы приложение не «падало», а продолжало работать или корректно сообщало пользователю о проблеме.

### Понятие исключений и стандартное поведение браузера

Когда JavaScript встречает ошибку во время выполнения (runtime error), выполнение скрипта немедленно прерывается. Все инструкции, идущие после строки с ошибкой, игнорируются, а в консоль браузера выводится сообщение о сбое. Такое поведение неприемлемо для качественного интерфейса: если одна кнопка вызвала ошибку, всё остальное приложение (например, навигация или формы) не должно переставать работать.

Для управления такими сценариями используется конструкция `try...catch`. Она позволяет «попробовать» выполнить блок кода и, если в нем возникнет ошибка, «перехватить» ее, не останавливая выполнение всего приложения.

### Конструкция try...catch

Синтаксис состоит из двух основных блоков: `try`, где мы пишем потенциально опасный код, и `catch`, который сработает только в случае возникновения ошибки.

```javascript
try {
  // Код, который мы хотим выполнить
  console.log("Начало проверки");
  
  // Пример ошибки: обращение к несуществующей переменной
  let result = dataFromAPI + 10; 
  
  console.log("Этот текст не выведется, если выше ошибка");
} catch (error) {
  // Этот блок сработает только при ошибке в try
  console.log("Произошла ошибка!");
  console.error("Детали:", error.message);
}

console.log("Приложение продолжает работу");
```

В блоке `catch` нам доступен объект ошибки (традиционно называемый `error` или `e`). Этот объект содержит полезные свойства:

* name: тип ошибки (например, `ReferenceError` или `TypeError`).
* message: текстовое описание проблемы.
* stack: подробная информация о том, в каком файле и на какой строке произошел сбой.

### Ограничения try...catch

Важно понимать, что `try...catch` работает только с синхронным кодом и только с синтаксически верным кодом.

1. Ошибки парсинга: Если вы забудете закрыть фигурную скобку или допустите другую опечатку в синтаксисе, скрипт не запустится вовсе, и `try...catch` не поможет.
2. Асинхронный код: Ошибки внутри функций вроде `setTimeout` не будут пойманы внешним блоком `try...catch`, так как основной поток кода уже завершится к моменту выполнения таймера. Чтобы поймать такую ошибку, `try...catch` должен находиться внутри самой асинхронной функции.

### Генерация собственных ошибок: оператор throw

Иногда JavaScript не видит проблемы там, где она есть с точки зрения бизнес-логики. Например, если сервер прислал объект пользователя без имени, технически это валидный объект, но для вашего приложения это «ошибка». В таких случаях мы используем оператор `throw`.

С помощью `throw` можно «выбросить» исключение вручную. Рекомендуется использовать для этого встроенный класс `Error`.

```javascript
let json = '{ "age": 25 }'; // Данные без имени

try {
  let user = JSON.parse(json);

  if (!user.name) {
    throw new Error("Данные неполны: имя обязательно");
  }

  console.log(user.name);
} catch (error) {
  console.log("Ошибка валидации: " + error.message);
}
```

Использование `new Error("сообщение")` создает объект с правильной структурой (свойствами name и message), что делает обработку более профессиональной и предсказуемой.

### Блок finally

Существует третий, опциональный блок — `finally`. Код внутри него выполнится в любом случае: завершился ли блок `try` успешно или произошла ошибка, обработанная в `catch`. Это идеальное место для завершающих действий, например, для скрытия индикатора загрузки (spinner) на странице.

```javascript
try {
  // Запрос данных
} catch (e) {
  // Обработка ошибки
} finally {
  // Убрать лоадер с экрана
  console.log("Завершение процесса");
}
```

### Итог

Изучение обработки ошибок позволяет создавать отказоустойчивые приложения. Конструкция `try...catch...finally` дает контроль над ходом выполнения программы, объект `Error` предоставляет информацию для отладки, а оператор `throw` позволяет гибко реагировать на нелогичные данные. Правильное использование этих инструментов превращает «падение» приложения в контролируемое событие, понятное и разработчику, и пользователю.

### Задача на закрепление

Напишите функцию `processData(jsonString)`, которая принимает строку в формате JSON.

1. Внутри функции используйте `try...catch`.
2. Попробуйте распарсить строку с помощью `JSON.parse()`.
3. Если в объекте отсутствует свойство `id`, выбросьте собственную ошибку `new Error("ID отсутствует")`.
4. В блоке `catch` выведите в консоль сообщение: "Ошибка в данных: \[текст ошибки]".
5. В блоке `finally` выведите "Обработка завершена".

**Критерии успеха:**

* Функция не вызывает фатального падения скрипта при передаче некорректного JSON или объекта без ID.
* В консоли корректно отображаются кастомные сообщения об ошибках.
* Блок `finally` срабатывает при любом вводе.
