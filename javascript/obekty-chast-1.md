# Объекты, часть 1

В то время как примитивные типы данных (строки, числа, булевы значения) хранят в себе только одно конкретное значение, объекты позволяют группировать связанные данные и функциональность в единую структуру. Для фронтенд-разработчика понимание работы с объектами критически важно, так как почти всё в JavaScript — от элементов DOM-дерева до ответов от сервера — представляется в виде объектов.

### Сущность и синтаксис объектов

Объект представляет собой неупорядоченную коллекцию свойств. Каждое свойство состоит из пары «ключ: значение». Ключ (или имя свойства) всегда является строкой, а значением может быть любой тип данных: от простой строки до другого объекта или функции. Самый распространенный способ создания объекта — использование фигурных скобок, что называется литералом объекта.

```javascript
const user = {
  name: "Александр",
  age: 28,
  "is developer": true
};
```

Внутри объекта свойства перечисляются через запятую. Если имя свойства содержит пробелы или дефисы, его необходимо оборачивать в кавычки. В современных стандартах JavaScript принято оставлять «висячую запятую» после последнего свойства — это упрощает редактирование кода и добавление новых полей в будущем.

### Чтение и изменение свойств

Для взаимодействия со свойствами объекта существуют два основных подхода: точечная нотация и квадратные скобки. Точечная нотация (`user.name`) наиболее лаконична и используется в большинстве случаев. Однако, если имя свойства хранится в переменной или содержит спецсимволы, на помощь приходят квадратные скобки.

```javascript
// Чтение свойств
console.log(user.name); // Точечная нотация
console.log(user["is developer"]); // Квадратные скобки для имен с пробелами

// Добавление и изменение
user.city = "Москва"; // Добавление нового свойства
user.age = 29; // Изменение существующего
```

Если свойство внутри объекта является функцией, оно называется методом. Вызов метода происходит так же, как и обращение к обычному свойству, но с добавлением круглых скобок в конце. Важно помнить, что при обращении к несуществующему свойству JavaScript не выдаст ошибку, а вернет значение `undefined`.

### Работа с константными объектами

Часто возникает вопрос: почему можно изменять свойства объекта, если он объявлен через `const`? Дело в том, что `const` защищает от переопределения саму переменную (ссылку на область памяти), но не запрещает модифицировать содержимое объекта, на который эта переменная указывает. Вы можете добавлять, удалять или изменять ключи, но не можете присвоить переменной `user` совершенно новый объект или другое значение.

### Полезные возможности: сокращенная запись и вычисляемые свойства

JavaScript предоставляет синтаксический сахар для создания объектов. Если имя переменной совпадает с будущим именем ключа объекта, можно использовать сокращенную запись.

```javascript
const role = "Admin";
const permissions = {
  role, // То же самое, что role: role
  createdAt: new Date()
};
```

Также имена свойств могут быть вычисляемыми. Если обернуть выражение в квадратные скобки прямо внутри литерала объекта, результатом выполнения этого выражения станет имя ключа.

### Проверка существования и перебор свойств

Иногда необходимо узнать, содержит ли объект определенное поле. Для этого используется оператор `in`. Он возвращает `true`, даже если значение свойства установлено в `undefined`, что делает его более надежным, чем простая проверка на равенство с `undefined`.

Для обхода всех свойств объекта применяется специальный цикл `for...in`. В ходе каждой итерации переменная цикла принимает значение ключа текущего свойства.

```javascript
for (let key in user) {
  console.log(`Ключ: ${key}, Значение: ${user[key]}`);
}
```

Стоит учитывать порядок свойств: ключи, которые могут быть интерпретированы как целые числа, всегда сортируются по возрастанию. Все остальные свойства располагаются в порядке их добавления в объект.

### Итог

Объекты позволяют структурировать информацию и описывать сложные сущности реального мира в коде. Мы изучили способы создания объектов, научились выполнять базовые операции (чтение, добавление, удаление через оператор `delete`), узнали о различиях в доступе к полям и механике работы цикла `for...in`. Эти знания являются базой для изучения более сложных тем, таких как прототипное наследование и деструктуризация.

### Задача на закрепление

Создайте объект `laptop`, который описывает характеристики ноутбука.

1. Объект должен содержать свойства: `brand` (строка), `model` (строка), `ram` (число), `isGaming` (булево значение).
2. Добавьте в объект метод `getSpecs`, который при вызове выводит в консоль строку с брендом и моделью.
3. Добавьте свойство `storage` через точечную нотацию после создания объекта.
4. Используя цикл `for...in`, выведите в консоль все ключи и их значения из этого объекта.
5. Удалите свойство `isGaming` и проверьте его отсутствие в объекте с помощью оператора `in`.

**Критерии успешного выполнения:**

* В консоли корректно отображаются все свойства объекта через цикл.
* Метод `getSpecs` успешно вызывается и выводит ожидаемую строку.
* После удаления оператор `in` возвращает `false` для свойства `isGaming`.
