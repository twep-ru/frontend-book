# Свойства и атрибуты DOM-элементов

Когда браузер читает HTML-документ, он создает соответствующие DOM-объекты для каждого тега. У этих объектов есть огромное количество встроенных свойств. Многие из них напрямую связаны с HTML-атрибутами: например, атрибут `action` у тега `<form>` становится свойством `action` у соответствующего объекта в JavaScript.

Работа со стандартными свойствами происходит по принципу геттеров и сеттеров. Если мы обращаемся к свойству `form.action`, мы получаем его текущее значение. Если же мы присваиваем этому свойству новую строку, изменения мгновенно отражаются в DOM-дереве и отображаются во вкладке Elements инструментов разработчика. Важно помнить, что если имя атрибута в HTML пишется через дефис (например, `aria-label`), то в JavaScript оно преобразуется в CamelCase (`ariaLabel`).

### Методы для работы с атрибутами

Не все атрибуты автоматически синхронизируются со свойствами объекта. Для работы с нестандартными или кастомными атрибутами существуют специальные методы, которые позволяют более гибко управлять состоянием элемента:

* `getAttribute(name)` — возвращает значение указанного атрибута в виде строки.
* `setAttribute(name, value)` — устанавливает значение атрибута (важно: значение всегда приводится к строке).
* `hasAttribute(name)` — проверяет наличие атрибута у элемента и возвращает логическое значение.
* `removeAttribute(name)` — полностью удаляет атрибут из элемента.

Использование этих методов необходимо, когда вы работаете с атрибутами, для которых нет прямых свойств в DOM-объекте, или когда вам нужно гарантированно взаимодействовать именно с атрибутом в разметке, а не с его представлением в памяти.

### Особенности синхронизации и исключения

В большинстве случаев изменения свойства и атрибута синхронизированы: меняете одно — обновляется другое. Однако существует важное исключение, которое часто путает новичков — это свойство `value` у полей ввода (`<input>`).

Атрибут `value` в HTML задает начальное значение поля. Если изменить это значение программно через свойство `element.value`, атрибут в разметке может не обновиться, хотя визуально текст в поле изменится. Это односторонняя синхронизация: изменение атрибута через `setAttribute` обновит свойство, но изменение свойства не всегда обновляет атрибут в HTML. Поэтому при работе с формами рекомендуется всегда обращаться напрямую к свойству `value` для получения актуальных данных, введенных пользователем.

### Data-атрибуты и передача данных

Для хранения кастомных данных в HTML используются `data-*` атрибуты. В JavaScript к ним можно получить доступ двумя путями: через универсальный метод `getAttribute` или через специальное свойство `dataset`. В объекте `dataset` все ключи представлены в CamelCase (например, `data-user-id` станет `userId`).

Иногда возникает необходимость передать сложную конфигурацию (целый объект) через атрибут. Для этого данные преобразуются в JSON-строку и помещаются в атрибут (часто используя одинарные кавычки снаружи, чтобы не конфликтовать с двойными кавычками внутри JSON). Чтобы использовать такие данные в коде, полученную строку нужно обработать методом `JSON.parse()`.

### Итог

Взаимодействие с DOM-элементами строится на понимании связи между HTML-атрибутами и свойствами объектов JavaScript. Хотя большинство стандартных атрибутов синхронизируются автоматически, для работы с кастомными данными следует использовать методы `getAttribute`/`setAttribute` или свойство `dataset`. Особое внимание стоит уделять полям ввода, где свойство `value` имеет свои нюансы поведения, отличные от обычных атрибутов.

### Задача на закрепление

Создайте небольшую форму с одним полем ввода и кнопкой. Выполните следующие действия:

1. Найдите поле ввода в DOM и установите ему кастомный атрибут `data-status` со значением `"active"`.
2. Добавьте обработчик события на кнопку, который при клике:
   * Считывает текущее значение из `input.value`.
   * Проверяет наличие атрибута `data-status` с помощью `hasAttribute`.
   * Если атрибут есть, выводит его значение в консоль и затем удаляет его через `removeAttribute`.
3. Попробуйте изменить значение атрибута `data-status` на объект в формате JSON (например, `{"id": 1}`) и считайте его в коде, преобразовав обратно в объект JavaScript.

**Критерии успешного выполнения:**

* Атрибут `data-status` успешно добавляется и удаляется из HTML-разметки (проверьте через панель Elements).
* Значение `value` корректно считывается независимо от того, как оно было введено.
* JSON-данные успешно парсятся и выводятся в консоль как объект, а не как строка.
