# Асинхронность

Понимание того, как код может выполняться «в фоновом режиме» без заморозки интерфейса, критически важно для создания отзывчивых веб-приложений. В этом уроке мы разберем внутренние механизмы работы движка JavaScript, научимся управлять очередностью задач и изучим современные инструменты для написания чистого асинхронного кода.

### Однопоточность и событийный цикл Event Loop

JavaScript является однопоточным языком программирования. Это означает, что в конкретный момент времени движок (например, V8 в Chrome) может выполнять не более одной строки кода. Код выполняется последовательно, строка за строчкой — такой принцип называется синхронным. Однако, если одна из функций будет выполняться слишком долго (например, тяжелые вычисления или ожидание ответа от сервера), она заблокирует выполнение всего последующего кода, и страница просто «зависнет».

Для решения этой проблемы существует событийный цикл — Event Loop. Его задача заключается в том, чтобы следить за основным потоком выполнения и очередью асинхронных задач. Когда движок встречает асинхронную операцию (например, `setTimeout`), он не ждет её завершения, а передает выполнение среде окружения (браузеру) и переходит к следующей строке. Как только асинхронная задача готова к выполнению, она попадает в очередь задач, и Event Loop переносит её в основной поток только тогда, когда тот полностью освободится.

### Проблема Callback Hell и появление Promises

Исторически асинхронность реализовывалась через функции обратного вызова или коллбэки. Коллбэк — это функция, которая передается в другую функцию и вызывается по завершении какого-либо события. Проблема возникает, когда результат одной асинхронной операции нужен для запуска следующей, что приводит к огромной вложенности кода, известной как Callback Hell (ад коллбэков). Такой код крайне сложно читать и поддерживать.

Для решения этой проблемы в JavaScript были введены промисы (Promises) — специальные объекты, представляющие результат будущей асинхронной операции. Промис всегда находится в одном из трех состояний:

* Pending (ожидание) — начальное состояние.
* Fulfilled (выполнено успешно) — операция завершена удачно.
* Rejected (выполнено с ошибкой) — произошел сбой.

```javascript
const myPromise = new Promise((resolve, reject) => {
  const success = true;
  setTimeout(() => {
    if (success) {
      resolve("Данные получены");
    } else {
      reject("Ошибка сервера");
    }
  }, 2000);
});

myPromise
  .then(data => console.log(data))  // Обработка успеха
  .catch(err => console.error(err)) // Обработка ошибки
  .finally(() => console.log("Завершено")); // Выполнится в любом случае
```

Промисы позволяют выстраивать плоские цепочки вызовов через метод `.then()`, где каждый следующий шаг ждет завершения предыдущего, что делает код значительно чище.

### Современный стандарт: Async/Await

Синтаксис Async/Await — это «синтаксический сахар» над промисами, который позволяет писать асинхронный код так, будто он синхронный. Ключевое слово `async` перед функцией гарантирует, что она всегда вернет промис. Ключевое слово `await` заставляет интерпретатор приостановить выполнение функции до тех пор, пока промис не разрешится, и только потом вернуть результат.

```javascript
async function fetchData() {
  try {
    const response = await makeRequest("https://api.example.com/data");
    console.log("Результат:", response);
  } catch (error) {
    console.error("Произошла ошибка:", error);
  }
}
```

Использование `try...catch` вместе с `async/await` позволяет обрабатывать ошибки максимально прозрачно, объединяя логику обработки синхронных и асинхронных исключений в единую структуру. Важно помнить, что `await` блокирует выполнение только внутри текущей `async` функции, не останавливая основной поток приложения.

### Итог

Асинхронность в JavaScript позволяет эффективно обрабатывать запросы и события без блокировки интерфейса благодаря механизму Event Loop. Мы изучили эволюцию подходов: от вложенных коллбэков к мощным объектам Promise и максимально лаконичному синтаксису Async/Await, который на сегодняшний день является стандартом индустрии для работы с сетевыми запросами и таймерами.

### Задача на закрепление

Напишите небольшую программу для имитации системы заказов в кафе.

1. Создайте функцию `prepareCoffee`, которая возвращает `Promise`. Функция должна имитировать приготовление кофе в течение 3 секунд (используйте `setTimeout`). При успешном завершении она должна возвращать строку "Кофе готов!", а при ошибке (имитируйте её с вероятностью 20%) — "Кофемашина сломалась".
2. Напишите асинхронную функцию `orderProcess`, используя `async/await`.
3. Внутри `orderProcess` выведите в консоль "Заказ принят, начинаем готовить...".
4. Вызовите `prepareCoffee` и дождитесь результата. Используйте `try...catch` для обработки возможной поломки.
5. В конце, независимо от результата, выведите в консоль "Хорошего дня!".

**Критерии успешного выполнения:**

* В консоли соблюдается верная последовательность сообщений с учетом задержки в 3 секунды.
* Используется синтаксис `async/await` для работы с промисом.
* Ошибки обрабатываются корректно через блок `catch`.
