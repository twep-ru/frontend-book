# FormData

Будь то простая подписка на рассылку или сложная многоэтапная регистрация, нам необходимо собрать введенные пользователем данные, обработать их и отправить на сервер. Понимание того, как эффективно извлекать информацию из полей ввода, позволяет создавать гибкие и контролируемые интерфейсы, выходящие за рамки стандартного поведения HTML-форм.

### Обработка события отправки

Прежде чем приступать к сбору данных, важно научиться контролировать сам процесс отправки формы. По умолчанию при нажатии на кнопку отправки (типа `submit`) браузер пытается перезагрузить страницу или перейти по адресу, указанному в атрибуте `action`. В современной разработке это чаще всего нежелательно, так как мы хотим отправить данные асинхронно.

Для предотвращения стандартного поведения используется метод `event.preventDefault()` внутри обработчика события `submit`. Это «замораживает» форму, позволяя нам программно считать все значения и выполнить валидацию или отправку через API. Код подписки на событие обычно выглядит так:

```javascript
const formElement = document.querySelector('.my-form');

formElement.addEventListener('submit', (event) => {
  event.preventDefault(); // Останавливаем перезагрузку страницы
  // Здесь будет логика сбора данных
});
```

### Эволюция сбора данных: от циклов к FormData

Исторически разработчики собирали данные, перебирая элементы формы (свойство `elements`) в цикле. Такой подход требует ручной проверки типов полей: например, для текстовых полей нужно брать `value`, для чекбоксов — проверять состояние `checked`, а для радиокнопок — искать только выбранный вариант. Это приводит к громоздкому коду с множеством условий и проверок на наличие атрибута `name`.

Современный стандарт JavaScript предлагает класс `FormData`, который автоматизирует этот процесс. Конструктор `new FormData(formElement)` при вызове сканирует форму и автоматически собирает все пары «имя поля: значение». Важное условие: у каждого интерактивного элемента в HTML (input, select, textarea) обязательно должен быть заполнен атрибут `name`, иначе `FormData` его проигнорирует.

```javascript
const formData = new FormData(formElement);
```

Объект, который мы получаем, является специальной коллекцией. Если вы попробуете вывести его в консоль напрямую, вы увидите пустой объект, так как его содержимое скрыто во внутренних механизмах. Однако данные там есть, и их можно извлечь или просмотреть.

### Методы работы и просмотр содержимого

Для отладки и преобразования данных в привычный формат объекта используется метод `Object.fromEntries()`. Он принимает итерируемый объект `FormData` и превращает его в стандартный JavaScript-объект, где ключами становятся имена полей. Это крайне удобно для последующей отправки данных на сервер в формате JSON.

Класс `FormData` также предоставляет набор встроенных методов для манипуляции данными «на лету» до их отправки:

* append(key, value) — позволяет добавить новое поле в коллекцию вручную.
* get(key) — возвращает значение конкретного поля по его имени.
* has(key) — проверяет, существует ли в форме поле с таким именем.
* delete(key) — удаляет поле из коллекции.

Эти инструменты дают разработчику полный контроль: вы можете программно добавить техническую информацию (например, ID сессии или временную метку) или удалить конфиденциальные данные перед тем, как они покинут клиентскую часть приложения.

### Итог

Использование класса `FormData` значительно упрощает жизнь разработчика, избавляя от необходимости писать сложные циклы для обхода элементов формы. Этот подход автоматически обрабатывает различные типы полей, такие как текстовые вводы, радиокнопки и файлы, объединяя их в удобную структуру данных. В сочетании с методом `event.preventDefault()` и преобразованием через `Object.fromEntries()`, этот инструмент становится стандартом для современной работы с пользовательским вводом в вебе.

### Задача на закрепление

Создайте функционал для формы обратной связи.

1. Создайте в HTML форму с полями: `userName` (текст), `userEmail` (email) и `message` (textarea). Добавьте кнопку отправки.
2. В JavaScript добавьте обработчик события `submit`, который предотвращает перезагрузку страницы.
3. Используя класс `FormData`, соберите все данные из формы.
4. Добавьте в собранные данные новое поле `timestamp` с текущей датой и временем с помощью метода `append`.
5. Преобразуйте результат в обычный объект и выведите его в консоль.
6. Удалите поле `userEmail` из объекта `FormData` и проверьте его отсутствие с помощью метода `has`, выведя результат проверки в консоль.

**Критерии успешного выполнения:**

* Страница не перезагружается при нажатии на кнопку.
* В консоли отображается объект со всеми полями формы и добавленным временем.
* Метод `has` возвращает `false` после удаления поля email.
