# Валидация форм

Использование стандартных HTML-атрибутов в сочетании с мощными инструментами JavaScript позволяет создавать гибкие и доступные интерфейсы, которые мгновенно реагируют на ошибки ввода.

### Нативная валидация и её ограничения

Современные браузеры предоставляют встроенные механизмы проверки данных через такие атрибуты, как `required`, `pattern`, `minlength` и `maxlength`. Однако стандартные браузерные сообщения (тултипы) сложно стилизовать, а их текст не всегда информативен для пользователя. Чтобы получить полный контроль над визуальным отображением ошибок, разработчики часто используют атрибут `novalidate` на уровне формы, что отключает стандартные уведомления и позволяет реализовать собственную логику на JavaScript.

### Свойство validity и объект ValidityState

Ключевым инструментом для создания кастомной валидации является свойство `validity`, доступное у каждого элемента ввода (`input`, `select`, `textarea`). Оно возвращает объект типа `ValidityState`, который содержит булевы флаги для различных типов ошибок:

* `valueMissing`: истинно, если поле с атрибутом `required` пустое.
* `tooShort`: сигнализирует о том, что введенное значение короче указанного в `minlength`.
*   `patternMismatch`: активируется, когда данные не соответствуют регулярному выражению в атрибуте `pattern`.

    Использование этого объекта позволяет избежать написания громоздких проверок вручную, так как браузер сам вычисляет состояние поля на основе его атрибутов.

### Архитектура компонента валидации

Для реализации масштабируемого решения удобно использовать классовый подход. Создание класса `FormsValidation` позволяет инкапсулировать логику проверки, сопоставление типов ошибок с человекопонятными сообщениями и управление DOM-элементами.

```javascript
class FormsValidation {
  constructor() {
    this.selectors = {
      form: '[data-js-form]',
      errors: '[data-js-form-field-errors]'
    };
    this.errorMessages = {
      valueMissing: () => 'Пожалуйста, заполните это поле',
      patternMismatch: ({ title }) => title || 'Данные не соответствуют формату',
      tooShort: ({ minLength }) => `Минимум символов: ${minLength}`
    };
    this.bindEvents();
  }
}
```

В таком компоненте сообщения об ошибках могут быть представлены в виде функций, которые динамически принимают свойства DOM-элемента (например, `title` или `minLength`) для формирования уточняющего текста.

### Обработка событий и фаза погружения

Валидацию отдельного поля логично проводить в момент потери им фокуса — по событию `blur`. Важной особенностью является то, что событие `blur` не всплывает по дереву DOM. Чтобы эффективно отслеживать его на уровне всего документа или формы, необходимо использовать фазу погружения (capturing), установив параметр `capture: true` в методе `addEventListener`. Для полей типа `radio` или `checkbox` проверку лучше выполнять по событию `change`, чтобы пользователь сразу видел результат выбора.

### Доступность и визуализация ошибок

При обнаружении ошибок важно не только вывести текст на экран, но и оповестить пользователей скринридеров. Для этого используется атрибут `aria-invalid`, значение которого (`true` или `false`) должно обновляться программно при каждой проверке. Сами сообщения об ошибках группируются в специальные контейнеры, связанные с полями ввода через идентификаторы. При отправке формы (событие `submit`) необходимо провести финальную проверку всех обязательных полей, предотвратить отправку в случае ошибок через `event.preventDefault()` и автоматически перевести фокус на первый невалидный элемент для удобства исправления.

### Итог

Изучение механизмов валидации через свойство `validity` позволяет создавать профессиональные формы с качественным пользовательским интерфейсом. Комбинируя нативные HTML-атрибуты для задания правил и JavaScript для управления состоянием и доступностью, разработчик получает надежный инструмент, который легко адаптируется под любые требования проекта и обеспечивает корректную работу с данными.

### Задача на закрепление

Создайте упрощенный компонент валидации для формы регистрации.

1. Форма должна содержать поля: `username` (обязательное, минимум 4 символа) и `email` (обязательное, тип email).
2. Реализуйте класс, который при потере фокуса (`blur`) проверяет поле на наличие ошибок `valueMissing` и `tooShort`.
3. Ошибки должны выводиться в текстовый блок под соответствующим инпутом.
4. Добавьте обработку события `submit`: если в форме есть ошибки, отправка должна блокироваться, а невалидное поле — подсвечиваться (например, добавлением CSS-класса).

**Критерии успешного выполнения:**

* При вводе менее 4 символов в поле `username` и уходе с него появляется сообщение об ошибке.
* При попытке отправить пустую форму страница не перезагружается, а под полями появляются уведомления.
* Код структурирован в виде класса или набора функций с использованием свойства `validity`.
