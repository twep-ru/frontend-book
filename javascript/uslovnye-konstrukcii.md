# Условные конструкции

Если примитивные типы данных и объекты — это «материя» программы, то условия — это её «мозг» и логика. Без них создание интерактивных интерфейсов, таких как валидация форм или переключение тем оформления, было бы невозможным.

### Условные конструкции: if, else и else if

Основным инструментом ветвления в JavaScript является инструкция `if`. Она вычисляет выражение в круглых скобках и приводит его к логическому (булевому) типу. Если результат равен `true`, выполняется блок кода в фигурных скобках. Для обработки ситуаций, когда условие ложно, используется необязательный блок `else`, который переводится как «иначе».

```javascript
const year = 2026;

if (year === 2026) {
  console.log("Текущий год!"); // Выполнится, если условие истинно
} else {
  console.log("Другой год"); // Выполнится, если условие ложно
}
```

Когда вариантов развития событий больше двух, на помощь приходит конструкция `else if`. Она позволяет выстраивать цепочки проверок любой длины. Программа будет проверять условия сверху вниз до первого совпадения, после чего проигнорирует все остальные блоки в этой цепочке.

### Тернарный оператор

Для лаконичной записи простых условий, когда нужно присвоить переменной одно из двух значений, идеально подходит тернарный оператор. Его название происходит от того, что он принимает три операнда: само условие, значение при истине и значение при лжи. Синтаксис выглядит как `условие ? значение1 : значение2`.

```javascript
const age = 20;
const status = age >= 18 ? "Взрослый" : "Ребенок";
```

Такой подход позволяет сразу делать переменные константными (`const`), избавляя от необходимости объявлять их через `let` заранее. Однако не стоит злоупотреблять вложенными тернарными операторами, так как это сильно усложняет чтение кода.

### Логические операторы: И, ИЛИ и НЕ

Для создания сложных условий используются логические операторы. Оператор «ИЛИ» (`||`) возвращает `true`, если хотя бы один из его операндов истинен. В более широком смысле он ищет и возвращает первое «истинное» значение слева направо. Если все значения ложны (например, `null`, `undefined` или `0`), он вернет последний операнд.

Оператор «И» (`&&`) работает строго наоборот: он возвращает `true` только в том случае, если оба операнда истинны. В цепочке вычислений он ищет первое «ложное» значение и возвращает его. Если все операнды истинны, результатом станет последний из них. Важно помнить, что приоритет у оператора «И» выше, чем у «ИЛИ», поэтому он всегда вычисляется первым.

Оператор «НЕ» (`!`) просто инвертирует булево значение: превращает `true` в `false` и наоборот. Двойное отрицание (`!!`) часто используется для быстрого приведения любого типа данных к булевому значению.

### Оператор нулевого слияния (??)

Относительно новый оператор `??` часто путают с оператором «ИЛИ». Главное различие в том, что «ИЛИ» считает ложными любые «пустые» значения (0, пустая строка, `false`), в то время как оператор нулевого слияния реагирует только на `null` и `undefined`.

```javascript
let count = 0;
let resultOR = count || 10; // resultOR будет 10, так как 0 — это false
let resultNullish = count ?? 10; // resultNullish будет 0, так как 0 — это не null/undefined
```

Это делает `??` незаменимым при работе с настройками по умолчанию, когда значения `0` или пустая строка являются валидными данными, которые нельзя заменять.

### Итог

Мы изучили основные способы управления логикой приложения: от базовых инструкций `if/else` до продвинутых операторов, таких как тернарный и оператор нулевого слияния. Понимание того, как работают логические «И» и «ИЛИ», а также знание их приоритетов, позволяет писать чистый и предсказуемый код, способный обрабатывать любые пользовательские сценарии.

### Задача на закрепление

Создайте систему проверки доступа для кинотеатра.

1. Объявите переменные: `userAge` (число), `hasTicket` (булево), `money` (число), `ticketPrice` (число).
2. Напишите условие: пользователь может пройти в зал, если ему исполнилось 18 лет И у него есть билет.
3. Добавьте проверку: если билета нет, но у пользователя достаточно денег (`money >= ticketPrice`), выведите сообщение «Вы можете купить билет».
4. Используйте тернарный оператор, чтобы записать в переменную `canEnter` строку «Доступ разрешен» или «Доступ запрещен» на основе условий выше.
5. Используя оператор `??`, выведите баланс пользователя, но если переменная `money` не задана (`null`), выведите «Баланс не определен».

**Критерии успешного выполнения:**

* В коде используются операторы `&&` или `||` для объединения условий.
* Тернарный оператор корректно возвращает строку в зависимости от возраста и наличия билета.
* Оператор `??` правильно отрабатывает ситуацию с отсутствующим значением баланса.
