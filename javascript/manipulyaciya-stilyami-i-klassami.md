# Манипуляция стилями и классами

Для фронтенд-разработчика крайне важно понимать, как управлять CSS через JavaScript, так как это лежит в основе создания анимаций, обработки состояний интерфейса (например, открытие модального окна или валидация формы) и реализации пользовательских тем. В этом уроке мы разберем основные инструменты работы со стилями: от прямого изменения свойств до управления CSS-переменными.

### Работа со свойством style

Самый прямой способ изменить оформление элемента — использовать свойство `style`. Каждый DOM-элемент обладает этим объектом, который содержит в себе все возможные CSS-свойства. Важно помнить, что в JavaScript названия свойств пишутся в формате camelCase (например, `backgroundColor` вместо `background-color`).

```javascript
const box = document.querySelector('.box');

// Изменение конкретных свойств
box.style.width = '200px';
box.style.backgroundColor = 'royalblue';
box.style.marginTop = '20px';
```

Свойство `style` работает исключительно с инлайновыми стилями (теми, что записываются в атрибут `style` самого HTML-тега). Если вам нужно задать сразу группу свойств, можно воспользоваться полем `cssText`. Однако будьте осторожны: прямая запись в `cssText` полностью перезаписывает все текущие инлайновые стили элемента. Чтобы этого избежать, используйте оператор дополнения `+=`.

```javascript
// Полная перезапись
box.style.cssText = 'width: 100px; height: 100px; color: white;';

// Добавление новых стилей к существующим
box.style.cssText += 'display: flex; justify-content: center;';
```

### Чтение вычисленных стилей через getComputedStyle

Часто возникает ситуация, когда нужно узнать ширину или цвет элемента, которые заданы не в атрибуте `style`, а во внешнем CSS-файле. Если вы попробуете прочитать `box.style.width` для элемента, стиль которого описан в файле `.css`, вы получите пустую строку.

Для получения финальных, реально примененных к элементу значений, используется глобальный метод `getComputedStyle()`. Он возвращает объект со всеми вычисленными свойствами, включая те, что были унаследованы или заданы внешними правилами.

```javascript
const styles = getComputedStyle(box);
console.log(styles.width); // Вернет актуальное значение, например '150px'

// Можно получить стили даже для псевдоэлементов
const beforeStyle = getComputedStyle(box, '::before');
console.log(beforeStyle.content);
```

### Управление классами: className и classList

Хотя прямое изменение стилей через `style` удобно для динамических вычислений (например, координат курсора), для оформления состояний лучше использовать CSS-классы. Это позволяет разделять логику и оформление. Свойство `className` позволяет работать со строкой классов целиком, но это не всегда удобно, так как легко случайно затереть существующие классы.

Современный и предпочтительный способ — объект `classList`. Он предоставляет набор методов для точечного управления классами:

* `add('class')` — добавляет один или несколько классов.
* `remove('class')` — удаляет указанные классы.
* `contains('class')` — проверяет наличие класса и возвращает `true` или `false`.
* `toggle('class')` — переключает класс: добавляет, если его нет, и удаляет, если он есть.

Метод `toggle` особенно полезен для реализации переключателей. У него есть второй необязательный аргумент — логическое условие. Если условие истинно, класс будет добавлен, если ложно — удален.

```javascript
const hasError = true;
box.classList.toggle('error-theme', hasError); // Класс добавится
```

### Динамическое управление CSS-переменными

CSS-переменные (custom properties) стали стандартом современной разработки. JavaScript позволяет изменять их значения на лету, что крайне удобно для реализации тем оформления или сложных визуальных эффектов. Для этого используется метод `setProperty`, который вызывается у объекта `style`.

```javascript
// Установка значения переменной
box.style.setProperty('--main-bg-color', '#ff5733');

// Можно также указать приоритет !important третьим аргументом
box.style.setProperty('border-width', '5px', 'important');
```

Изменяя значение одной переменной в родителе, вы можете мгновенно обновить внешний вид множества дочерних элементов, что делает этот подход очень производительным и элегантным.

### Итог

Манипуляция стилями в JavaScript строится вокруг нескольких ключевых инструментов: объекта `style` для быстрой правки инлайновых свойств, метода `getComputedStyle` для чтения финальных параметров и объекта `classList` для управления классами состояний. Использование `classList` и CSS-переменных считается наиболее чистым подходом, так как позволяет сохранять описание внешнего вида в CSS, оставляя за JavaScript только управление логикой переключения этих состояний.

### Задача на закрепление

Создайте интерактивную карточку товара, которая меняет свое состояние при взаимодействии.

1. Создайте в HTML блок с классом `card` и кнопкой внутри.
2. При клике на кнопку карточка должна переключать класс `card--active`, который меняет её цвет фона и размер шрифта (опишите эти стили в CSS).
3. Используя `style.setProperty`, реализуйте изменение цвета рамки карточки на случайный цвет при каждом клике.
4. Выведите в консоль текущую ширину карточки, полученную с помощью `getComputedStyle`, сразу после изменения её состояния.

**Критерии успешного выполнения:**

* Класс `card--active` корректно добавляется и удаляется при кликах через `classList.toggle`.
* Цвет рамки меняется динамически через обращение к инлайновым стилям или CSS-переменной.
* В консоли отображается актуальная ширина (в пикселях), а не пустая строка.
