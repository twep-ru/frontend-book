# Функции, часть 1

Для фронтенд-разработчика понимание этой темы критически важно, так как современная веб-разработка строится на переиспользовании логики, обработке событий и создании компонентов. Функции позволяют группировать инструкции, давать им понятные имена и выполнять этот код в любой нужный момент, что делает архитектуру приложения чистой и поддерживаемой.

### Основы объявления функций

Самый классический способ создать функцию в JavaScript называется Function Declaration. Синтаксис начинается с ключевого слова `function`, за которым следует имя (обычно в стиле camelCase), круглые скобки и фигурные скобки, ограничивающие тело функции.

```javascript
function greet() {
  console.log("Привет, мир!");
}
```

Важно понимать, что само по себе объявление функции не запускает код внутри неё. JavaScript просто запоминает, что такая функция существует. Чтобы инструкции выполнились, функцию необходимо «вызвать» (execute), написав её имя со скобками: `greet()`.

### Область видимости (Scope)

Область видимости определяет, где именно в коде доступны ваши переменные. В JavaScript существует глобальная и локальная (блочная) области видимости. Любая переменная, созданная внутри фигурных скобок функции, считается локальной. Это означает, что она «живет» только внутри этой функции и недоступна снаружи.

```javascript
function showMessage() {
  let message = "Локальное сообщение";
  console.log(message); // Выведет текст успешно
}

showMessage();
// console.log(message); // Ошибка: message is not defined
```

Если же переменная объявлена вне функции, она считается глобальной. Функции имеют доступ к внешним переменным, но внешняя среда не имеет доступа к тому, что спрятано внутри функции. Это защищает код от случайных конфликтов имен и ошибок.

### Параметры и аргументы

Чтобы функции были гибкими, они должны уметь принимать данные. Для этого используются параметры — это переменные, которые указываются в круглых скобках при объявлении функции. Когда мы вызываем функцию и передаем ей конкретные данные, эти данные называются аргументами.

```javascript
function logUser(name, count = 1) {
  for (let i = 0; i < count; i++) {
    console.log("Пользователь: " + name);
  }
}

logUser("Алексей", 3); // "Алексей" и 3 — это аргументы
```

В примере выше для параметра `count` задано значение по умолчанию (`1`). Если при вызове мы забудем передать второй аргумент, JavaScript не выдаст ошибку и не подставит `undefined`, а будет использовать указанную единицу. Это хорошая практика, делающая функции более устойчивыми к некорректным входным данным.

### Возврат значения: оператор return

Функции могут не только выполнять действия (выводить в консоль, менять цвет кнопки), но и возвращать результат своей работы обратно в то место, где они были вызваны. Для этого используется ключевое слово `return`.

Как только интерпретатор JavaScript встречает `return`, выполнение функции немедленно прекращается, и значение, указанное после этого слова, отдается «наружу». Если `return` не указан или после него ничего нет, функция по умолчанию вернет `undefined`.

```javascript
function sum(a, b) {
  return a + b;
  console.log("Этот код никогда не выполнится"); // Код после return игнорируется
}

let result = sum(5, 10); // Переменная result теперь хранит число 15
```

Часто `return` используют для раннего выхода из функции. Например, если входные данные не прошли проверку на валидность, можно сразу вернуть сообщение об ошибке, не нагружая браузер выполнением остальной логики.

### В этом уроке

В этом уроке мы разобрали базу работы с функциями: от их создания до механики передачи данных и возврата результата. Мы изучили, как область видимости изолирует код внутри функций, предотвращая хаос в глобальном пространстве имен, и как параметры по умолчанию делают наши инструменты более гибкими. Понимание этих основ позволяет переходить к более сложным темам, таким как асинхронность или создание сложных интерфейсных компонентов.

### Задача на закрепление

Напишите функцию `calculateFinalPrice`, которая принимает два параметра: `price` (исходная цена товара) и `discount` (процент скидки).

**Требования к выполнению:**

1. Если аргумент `discount` не передан, скидка по умолчанию должна составлять 0%.
2. Внутри функции добавьте проверку: если `price` не является числом или меньше нуля, функция должна вернуть строку: "Ошибка: неверная цена".
3. Функция должна возвращать итоговую цену (число) после вычета скидки.
4. Вызовите функцию несколько раз с разными значениями (включая ошибку и отсутствие скидки) и выведите результат в консоль.

**Критерии успеха:**

Функция корректно обрабатывает отсутствие аргумента, не «падает» при получении текста вместо числа и возвращает правильный математический результат.
