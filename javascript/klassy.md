# Классы

Если раньше разработчикам приходилось полагаться исключительно на функции-конструкторы и прототипы, то современный синтаксис классов делает код более структурированным, читаемым и удобным для поддержки. Понимание классов критически важно для фронтенд-разработки, так как эта концепция лежит в основе создания сложных компонентов интерфейса, таких как слайдеры, модальные окна или кастомные элементы управления.

### Сущность и базовый синтаксис классов

По своей сути класс — это многофункциональный шаблон или «калька», по которой создаются однотипные объекты. Вместо того чтобы описывать каждый объект вручную, вы один раз задаете правила в классе, а затем генерируете необходимое количество экземпляров.

Объявление класса начинается с ключевого слова `class`, за которым следует имя (принято использовать заглавную букву, например, `Student`) и фигурные скобки, ограничивающие тело класса. Для создания конкретного объекта на основе этого шаблона используется оператор `new`.

```javascript
class Student {
  // Тело класса
}

const firstStudent = new Student(); // Создание экземпляра класса
```

Хотя изначально созданный объект может быть пустым, сам класс служит фундаментом для всей будущей логики.

### Метод constructor и работа с данными

Почти каждый класс содержит специальный метод — `constructor`. Он уникален тем, что выполняется автоматически в момент создания нового экземпляра через `new`. Основная задача конструктора — инициализация объекта, то есть наполнение его начальными данными.

Внутри конструктора и других методов класса ключевое слово `this` ссылается на текущий создаваемый экземпляр. Это позволяет динамически присваивать свойствам объекта значения, переданные при его создании.

```javascript
class Student {
  constructor(name, age) {
    this.name = name;
    this.age = age;
  }

  logAge() {
    console.log(`Возраст студента: ${this.age}`);
  }
}

const vasya = new Student("Вася", 20);
vasya.logAge(); // Выведет: Возраст студента: 20
```

Важно помнить, что методы в классе пишутся без ключевого слова `function` и не разделяются запятыми, в отличие от обычных объектов. Если значение свойства известно заранее и не зависит от входных данных, его можно объявить в теле класса напрямую, минуя конструктор.

### Геттеры, сеттеры и инкапсуляция

Иногда нам нужно не просто записывать или читать значения свойств, а добавлять к этим действиям дополнительную логику. Для этого используются геттеры (`get`) и сеттеры (`set`). Геттер срабатывает при попытке прочитать свойство, а сеттер — при попытке присвоить ему значение. Это позволяет, например, автоматически форматировать текст или проверять корректность вводимых данных.

Для защиты внутренней логики класса от прямого вмешательства извне используется инкапсуляция. В современном JavaScript для создания по-настоящему приватных свойств и методов используется символ решетки `#` перед их именем. К таким сущностям нельзя обратиться напрямую извне класса — только через его внутренние методы.

```javascript
class Student {
  #city = "Москва"; // Приватное свойство

  get city() {
    return `г. ${this.#city}`;
  }

  set city(value) {
    this.#city = value.trim();
  }
}
```

Ранее разработчики использовали символ подчеркивания `_` в начале имени, чтобы пометить свойство как приватное, но это было лишь негласным правилом, которое не запрещало доступ технически.

### Статические свойства и наследование

В некоторых случаях свойства или методы должны принадлежать не конкретному объекту (экземпляру), а самому классу в целом. Для этого используется ключевое слово `static`. Статические методы удобно использовать для вспомогательных функций, которые связаны с тематикой класса, но не требуют данных от конкретного экземпляра.

Одной из самых мощных возможностей классов является наследование, которое реализуется с помощью ключевого слова `extends`. Оно позволяет создать новый класс на базе существующего, заимствуя все его свойства и методы. Чтобы расширить функционал родительского конструктора или метода, используется ключевое слово `super()`.

```javascript
class Person {
  constructor(name) {
    this.name = name;
  }
}

class Developer extends Person {
  constructor(name, language) {
    super(name); // Вызов конструктора родителя
    this.language = language;
  }
}
```

Это позволяет выстраивать иерархии объектов и избегать дублирования кода, переиспользуя логику базовых классов в более специализированных наследниках.

Классы помогают организовывать код в логические блоки, превращая разрозненные функции в структурированную систему. Использование конструкторов для инициализации, приватных полей для защиты данных и механизма наследования для расширения функционала делает разработку более предсказуемой и масштабируемой. Хотя многие задачи можно решить и без классов, их применение во фронтенде способствует созданию чистого и профессионального кода.

### Задача на закрепление

Создайте систему классов для управления транспортным средством.

1. Создайте базовый класс `Vehicle`. В конструкторе он должен принимать `brand` (бренд) и `maxSpeed` (максимальная скорость). Добавьте метод `move()`, который выводит в консоль: "{brand} движется со скоростью до {maxSpeed} км/ч".
2. Создайте класс-наследник `Car`, который расширяет `Vehicle`. Добавьте ему приватное свойство `#fuelLevel` (уровень топлива), изначально равное 100.
3. В классе `Car` реализуйте геттер для `#fuelLevel`, который возвращает строку "Топливо: {значение}%".
4. Добавьте в `Car` метод `drive()`, который вызывает родительский метод `move()` через `super`, а затем уменьшает уровень топлива на 10 и выводит текущий уровень.
5. Создайте экземпляр `Car`, вызовите метод `drive()` и попробуйте обратиться к приватным данным напрямую (убедитесь, что это вызовет ошибку).

**Критерии успешного выполнения:**

* Класс `Car` успешно наследует свойства и методы `Vehicle`.
* Метод `super()` корректно используется в конструкторе или методах наследника.
* Приватное свойство `#fuelLevel` недоступно вне класса.
* Геттер корректно форматирует вывод данных.
