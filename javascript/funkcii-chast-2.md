# Функции, часть 2

Понимание различных способов объявления функций и механик их работы — критически важный навык для фронтенд-разработчика, так как это напрямую влияет на читаемость, тестируемость и архитектуру веб-приложения.

### Способы объявления функций

В современном JavaScript существует три основных подхода к созданию функций, каждый из которых имеет свои синтаксические и поведенческие особенности.

#### **Function Declaration**

Это классический способ объявления, использующий ключевое слово `function`, за которым следует имя. Главная особенность этого метода — поднятие (hoisting). JavaScript интерпретирует код так, будто эти функции объявлены в самом начале их области видимости. Это позволяет вызывать функцию до того, как она фактически написана в коде.

```javascript
// Вызов возможен до объявления
console.log(sum(5, 5)); 

function sum(a, b) {
  return a + b;
}
```

#### **Function Expression**

Здесь функция создается как часть выражения и обычно присваивается переменной. В отличие от предыдущего способа, такие функции не участвуют в «поднятии». Вы не сможете вызвать их до строки, где они инициализированы. Это делает поток выполнения кода более предсказуемым и строгим.

```javascript
const multiply = function(a, b) {
  return a * b;
};

console.log(multiply(4, 3)); // 12
```

#### **Стрелочные функции (Arrow Functions)**

Появившись в стандарте ES6, стрелочные функции предложили более лаконичный синтаксис. Они особенно удобны для коротких операций благодаря возможности неявного возврата: если тело функции состоит из одного выражения и не обернуто в фигурные скобки, результат этого выражения возвращается автоматически. Важно помнить, что у стрелочных функций нет собственного контекста `this` и объекта `arguments`.

```javascript
// Краткая форма с неявным возвратом
const square = n => n * n;

// Обычная форма со скобками
const greet = (name) => {
  return `Привет, ${name}!`;
};
```

### Функции как значения и колбэки

В JavaScript функции считаются «объектами первого класса». Это означает, что их можно передавать в другие функции в качестве аргументов или возвращать из них. Функции, передаваемые как аргументы, называются колбэками (callback). Эта концепция лежит в основе асинхронного программирования и обработки событий во фронтенде.

Когда мы передаем функцию, мы указываем её имя без круглых скобок. Если добавить скобки, функция выполнится мгновенно, и передастся результат её выполнения, а не она сама.

```javascript
function processUser(name, callback) {
  console.log("Подготовка данных...");
  callback(name);
}

const showWelcome = (name) => console.log(`Добро пожаловать, ${name}`);

processUser("Алексей", showWelcome);
```

### Искусство именования и чистота кода

Имя функции должно быть глаголом, так как функция всегда выполняет какое-то действие. Хорошей практикой считается использование префиксов, которые сразу поясняют суть операции: `get` (получить), `show` (показать), `check` (проверить), `create` (создать). Например, имя `fetchData` гораздо информативнее, чем просто `data`.

При проектировании функций следует придерживаться принципа единственной ответственности: одна функция должна решать одну конкретную задачу. Если функция начинает делать слишком много (например, и вычислять данные, и обновлять интерфейс, и сохранять что-то в локальное хранилище), её стоит разбить на несколько мелких, специализированных функций. Это упрощает отладку и делает код человекочитаемым.

### Итог

Изучение различий между Function Declaration, Expression и стрелочными функциями позволяет выбирать наиболее эффективный инструмент для каждой задачи. Мы узнали, что функции могут гибко перемещаться по коду как обычные значения через колбэки и возвращаемые результаты. Правильное именование и соблюдение принципа одной задачи превращают сложный программный код в понятную и легко поддерживаемую структуру, что является признаком высокого профессионализма разработчика.

### Задача на закрепление

Напишите программу, которая имитирует обработку заказа в интернет-магазине.

1. Создайте стрелочную функцию `calculateTotal`, которая принимает цену товара и количество, возвращая их произведение (используйте неявный возврат).
2. Напишите функцию `checkout` (любым способом), которая принимает имя пользователя и колбэк-функцию.
3. Внутри `checkout` должна выводиться фраза: "Пользователь \[Имя] оформляет заказ...".
4. Вызовите `checkout`, передав в неё имя и функцию, которая выводит в консоль итоговую сумму (используя `calculateTotal`).

**Критерии успешного выполнения:**

* Использованы разные типы функций (стрелочная и классическая/выражение).
* Продемонстрирована передача колбэка.
* Код не содержит ошибок и выводит логически верные сообщения в консоль.
