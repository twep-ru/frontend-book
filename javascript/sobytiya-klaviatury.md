# События клавиатуры

Для фронтенд-разработчика понимание работы событий клавиатуры и ввода критически важно для создания удобных форм, настройки горячих клавиш (hotkeys) и валидации данных. В этом уроке мы разберем события `keydown`, `keyup`, `input`, `change`, а также работу с буфером обмена через события `cut`, `copy` и `paste`.

### События клавиатуры: keydown и keyup

События клавиатуры срабатывают при взаимодействии с физической или виртуальной клавиатурой. Событие `keydown` происходит в момент нажатия (вдавливания) клавиши, а `keyup` — в момент ее отпускания. Важно отметить, что эти события не предназначены для отслеживания ввода данных в текстовые поля. Их главная задача — обработка горячих клавиш (комбинаций для быстрого выполнения действий) в полноценных веб-приложениях.

Обычно обработчики этих событий вешают глобально на объект `document`, поскольку события клавиатуры всплывают по DOM-дереву. При срабатывании события в функцию-обработчик передается объект события `Event`, содержащий детальную информацию о нажатой клавише.

```javascript
document.addEventListener('keydown', (event) => {
  console.log(event);
});
```

В объекте события есть два ключевых свойства для идентификации клавиши: `key` и `code`. Свойство `key` содержит символ нажатой клавиши в зависимости от текущей раскладки клавиатуры (например, `q` на английской раскладке или `й` на русской). Свойство `code` содержит подкапотный код физической клавиши (например, `KeyQ`), который остается неизменным вне зависимости от выбранного языка. Для программирования горячих клавиш надежнее использовать именно `code`, чтобы комбинации работали корректно на любых раскладках.

Для обработки комбинаций клавиш (например, Shift + S) в объекте события предусмотрены булевы свойства: `altKey`, `ctrlKey`, `shiftKey` и `metaKey`. Они возвращают `true`, если соответствующая системная клавиша была зажата в момент возникновения события. Свойство `metaKey` отвечает за клавишу Command на устройствах с macOS.

### Отмена действий браузера по умолчанию

При нажатии на некоторые клавиши браузер выполняет встроенные действия. Например, клавиша `PageDown` прокручивает страницу вниз, а `Tab` переключает фокус между интерактивными элементами. Любое подобное поведение можно отменить с помощью метода `event.preventDefault()`.

```javascript
document.addEventListener('keydown', (event) => {
  if (event.code === 'Tab') {
    event.preventDefault(); // Отменяем стандартное переключение фокуса
  }
});
```

Этот механизм также полезен для ограничения ввода определенных символов в текстовые поля. Если мы хотим запретить ввод цифр в поле для имени, можно проверять нажимаемые клавиши с помощью регулярного выражения и отменять событие, если символ является цифрой. Для этого можно использовать регулярное выражение `/\d/`, метод `test` которого вернет `true`, если переданная строка содержит число.

### События ввода: input и change

Использование событий клавиатуры для отслеживания текста в полях ввода `input` или `textarea` является ошибкой. Код, привязанный к `keydown`, будет выполняться до того, как значение поля фактически обновится, что приведет к отставанию данных на один шаг. Событие `keyup` решает эту проблему, но не может отследить вставку текста через контекстное меню мыши (копировать/вставить), так как при этом не используются клавиши.

Для работы с вводом предназначено событие `input`. Оно мгновенно срабатывает при любом изменении содержимого текстового поля, независимо от того, как текст был изменен: с клавиатуры, через буфер обмена комбинациями клавиш или мышью.

```javascript
const inputElement = document.querySelector('.name-input');

inputElement.addEventListener('input', (event) => {
  console.log('Текущее значение:', event.target.value);
});
```

Однако событие `input` может быть слишком «агрессивным», если вы выполняете сложную валидацию или отправляете запросы на сервер. Вызов тяжелой логики на каждый введенный символ нерационален и может привести к зависаниям или излишней нагрузке. В таких случаях применяется событие `change`. Для текстовых полей оно срабатывает не при каждом изменении, а только в момент потери элементом фокуса (когда пользователь кликает вне поля или нажимает Tab), фиксируя окончательный результат ввода.

### События буфера обмена: cut, copy и paste

Иногда требуется перехватить действия пользователя, связанные с буфером обмена. Для этого существуют события `cut` (вырезание), `copy` (копирование) и `paste` (вставка). Они работают как при использовании клавиатурных сокращений, так и при выборе соответствующих пунктов в контекстном меню.

Для получения текста из буфера обмена рекомендуется использовать современный асинхронный API браузера `navigator.clipboard.readText()`. Он работает стабильно в различных обработчиках и возвращает Promise с актуальным текстовым содержимым буфера.

```javascript
document.addEventListener('paste', () => {
  navigator.clipboard.readText().then(text => {
    console.log('Текст из буфера:', text);
  });
});
```

### Итог

События клавиатуры (`keydown`, `keyup`) служат для отслеживания физических нажатий и программирования горячих клавиш, опираясь на неизменный код клавиши (`event.code`) и системные модификаторы (`event.shiftKey`, `event.ctrlKey`, `event.metaKey`). Для работы с формами и отслеживания вводимого текста используются события `input` (для мгновенной реакции на любые изменения, включая контекстное меню) и `change` (для действий после завершения ввода и потери фокуса), что позволяет корректно и без лишних срабатываний обрабатывать любые способы ввода.

### Задача на закрепление

Создайте простую форму с текстовым полем и элементом для вывода сообщения.

1. Создайте HTML-разметку с полем `<input type="text" id="username">` и пустым блоком `<div id="greeting"></div>`.
2. Используя JavaScript, повесьте обработчик события на поле ввода так, чтобы при любом изменении текста элемент `greeting` мгновенно обновлялся строкой `Привет, [введенное имя]!`.
3. Реализуйте защиту от ввода цифр: перехватывайте нажатия клавиш на уровне документа или самого инпута и отменяйте действие браузера по умолчанию, если вводимый символ является числом.
4. Добавьте валидацию: когда пользователь завершает ввод и убирает фокус с текстового поля, проверяйте длину текста. Если она меньше 3 символов, выводите в консоль предупреждение "Имя слишком короткое".

**Критерии успешного выполнения:**

* Текст в блоке приветствия обновляется без задержек при добавлении, удалении или вставке текста мышью.
* В поле невозможно ввести цифры ни с основной, ни с дополнительной цифровой клавиатуры.
* Сообщение об ошибке длины имени появляется только после того, как пользователь закончил ввод и кликнул мимо текстового поля.
