# JSON

Нам постоянно нужно обмениваться данными с сервером: отправлять информацию о пользователе или получать список товаров для интернет-магазина. Однако напрямую передать объект JavaScript через сеть невозможно, так как сетевые протоколы работают с текстом. Для решения этой проблемы был создан формат JSON (JavaScript Object Notation) — легкий и универсальный стандарт обмена данными, который понимает практически любой язык программирования.

### Сущность и строгие правила синтаксиса JSON

Хотя JSON внешне очень напоминает обычный литерал объекта в JavaScript, он является текстовым форматом и накладывает ряд жестких ограничений на структуру данных. Это необходимо для того, чтобы данные могли быть однозначно прочитаны любой системой, независимо от используемого языка программирования.

**Основные отличия и правила JSON включают в себя:**

* Обязательные двойные кавычки: В JSON ключи (имена свойств) и строковые значения должны быть обернуты только в двойные кавычки. Использование одинарных кавычек или обратных тиков приведет к ошибке парсинга.
* Типы данных: JSON поддерживает ограниченный набор типов: строки, числа, булевы значения (`true`/`false`), `null`, а также вложенные массивы и объекты.
* Никаких функций и undefined: В формате JSON нельзя хранить методы объекта (функции), символы или значение `undefined`. Если при преобразовании JavaScript-объекта в JSON в нем встретятся такие поля, они будут просто проигнорированы.
* Запрет на висячие запятые: В отличие от современного JavaScript, где запятая после последнего свойства объекта считается хорошим тоном, в JSON она строго запрещена. Наличие лишней запятой перед закрывающей скобкой сделает весь JSON-файл невалидным.

### Инструменты для работы с JSON в JavaScript

Для превращения живых объектов JavaScript в текстовый формат и обратно в языке предусмотрен глобальный объект `JSON`, предоставляющий два ключевых метода: `stringify` и `parse`.

#### Преобразование в строку: JSON.stringify()

Когда нам нужно подготовить данные для отправки на сервер, мы используем метод `JSON.stringify()`. Он берет JavaScript-объект (или массив) и «сериализует» его, то есть превращает в строку, соответствующую стандартам JSON.

```javascript
const user = {
  name: "Иван",
  age: 25,
  isAdmin: false,
  status: undefined, // Будет удалено
  sayHello() { console.log("Привет!"); } // Будет удалено
};

const jsonString = JSON.stringify(user);
console.log(jsonString); 
// Результат: '{"name":"Иван","age":25,"isAdmin":false}'
```

#### Преобразование в объект: JSON.parse()

Обратный процесс происходит, когда мы получаем данные от сервера в виде строки. Чтобы начать работать с ними как с обычным объектом (обращаться к свойствам через точку, менять значения), строку нужно «распарсить» с помощью метода `JSON.parse()`.

```javascript
const responseFromServer = '{"id": 101, "title": "Изучить JSON", "completed": true}';

const task = JSON.parse(responseFromServer);
console.log(task.title); // "Изучить JSON"
```

Важно помнить, что если строка содержит синтаксическую ошибку (например, пропущена кавычка или стоит лишняя запятая), метод `JSON.parse()` вызовет критическую ошибку в коде, которая может остановить работу всего приложения. Поэтому при работе с реальными данными извне этот процесс часто оборачивают в защитные конструкции.

### JSON как массив данных

Распространенным заблуждением является мнение, что JSON обязательно должен начинаться с фигурных скобок. На самом деле, верхнеуровневой структурой JSON вполне может быть массив. Это часто встречается в API, когда сервер возвращает не один объект, а целый список сущностей, например, массив объектов с данными о погоде или список последних постов в блоге. Такие данные обрабатываются теми же методами `stringify` и `parse` без каких-либо изменений в логике.

### Итог

JSON стал стандартом де-факто в веб-разработке благодаря своей лаконичности и простоте. Мы изучили, что это строго текстовый формат, требующий использования двойных кавычек и запрещающий хранение функций или использование висячих запятых. С помощью встроенных методов `JSON.stringify()` и `JSON.parse()` разработчики могут легко переводить программные структуры данных в текстовый вид для передачи по сети и восстанавливать их обратно для дальнейшей работы в коде.

### Задача на закрепление

Представьте, что вы разрабатываете систему управления библиотекой. Вам необходимо подготовить данные о книге для отправки в базу данных и обработать ответ от сервера.

1. Создайте объект `book` со следующими свойствами: `title` (строка), `author` (строка), `pages` (число), `isAvailable` (булево значение), а также добавьте метод `checkOut`, который просто выводит текст в консоль.
2. Преобразуйте этот объект в JSON-строку с помощью `JSON.stringify()` и выведите результат в консоль. Обратите внимание, что произошло с методом `checkOut`.
3. Создайте переменную `serverData`, содержащую строку: `'{"id": 5, "shelf": "B12", "tags": ["classic", "drama"]}'`.
4. Распарсите эту строку в JavaScript-объект.
5. Объедините данные из исходного объекта `book` и объекта, полученного из `serverData`, в один общий объект `fullBookInfo` и выведите его в консоль.

**Критерии успешного выполнения:**

* В консоли отображается валидная JSON-строка (без метода `checkOut`).
* Строка `serverData` успешно преобразована в объект, и вы можете получить доступ к его полям (например, `tags`).
* Итоговый объект `fullBookInfo` содержит информацию из обоих источников.
