# Браузерные хранилища данных

Понимание того, как работают браузерные хранилища, необходимо каждому фронтенд-разработчику для создания удобных интерфейсов: от сохранения состояния корзины в интернет-магазине до реализации темной темы, которая не сбрасывается при перезагрузке страницы.

### Понятие и виды браузерных хранилищ

В современной веб-разработке существует несколько механизмов для хранения информации в браузере пользователя. Основное различие между ними заключается в объеме доступной памяти, времени жизни данных и способе взаимодействия с сервером. Мы сосредоточимся на трех наиболее часто используемых: Cookies, LocalStorage и SessionStorage.

### Работа с Cookies

Cookies (куки) — это небольшие фрагменты текстовых данных, которые привязаны к конкретному домену. Главная особенность кук в том, что они автоматически отправляются браузером на сервер при каждом HTTP-запросе. Это делает их идеальным инструментом для аутентификации (хранения токенов сессии), но накладывает строгие ограничения: объем одной записи не может превышать 4 КБ, а их общее количество на домен ограничено (обычно до 20–50 штук).

Для работы с куками в JavaScript используется свойство `document.cookie`. Оно ведет себя специфически: при чтении возвращает строку со всеми куками через точку с запятой, а при записи — позволяет добавлять или обновлять только одну конкретную запись.

```javascript
// Запись куки с указанием имени и значения
document.cookie = "username=Alexander";

// Чтение всех кук (вернет строку вида "key1=val1; key2=val2")
console.log(document.cookie);
```

Для корректной работы со спецсимволами (например, пробелами) значения рекомендуется оборачивать в функцию `encodeURIComponent()`, а для удаления записи — устанавливать ей отрицательный срок жизни через параметр `max-age=-1`.

### LocalStorage и SessionStorage: Web Storage API

Эти хранилища гораздо удобнее для фронтенд-разработчика, так как они предоставляют простой программный интерфейс и позволяют хранить до 5 МБ данных, которые не перегружают сетевые запросы к серверу.

1. LocalStorage: данные сохраняются бессрочно. Они переживают перезагрузку страницы и даже закрытие браузера.
2. SessionStorage: данные живут только в рамках текущей вкладки. Как только пользователь закрывает вкладку, хранилище очищается.

Оба хранилища используют идентичные методы для манипуляции данными:

* `setItem(key, value)` — сохраняет данные. Важно помнить, что значения всегда приводятся к строке.
* `getItem(key)` — получает данные по ключу.
* `removeItem(key)` — удаляет конкретную запись.
* `clear()` — полностью очищает хранилище.

Если вам нужно сохранить сложный объект или массив, используйте формат JSON:

```javascript
const user = { name: "Ivan", age: 25 };

// Сохранение объекта
localStorage.setItem("user", JSON.stringify(user));

// Получение и парсинг обратно в объект
const savedUser = JSON.parse(localStorage.getItem("user"));
```

### Практическое применение: Смена темы сайта

Использование `localStorage` — стандарт индустрии для сохранения настроек интерфейса. Логика проста: при клике на переключатель мы добавляем класс (например, `is-dark-theme`) на элемент `html` и одновременно записываем текущее состояние в хранилище. При следующей загрузке страницы скрипт первым делом проверяет `localStorage` и, если там сохранено значение "dark", мгновенно применяет нужный класс, предотвращая "мигание" интерфейса.

### IndexedDB: База данных в браузере

Для очень сложных задач, требующих хранения больших объемов структурированных данных (например, для работы приложения в офлайн-режиме), существует IndexedDB. Это полноценная транзакционная база данных. В отличие от простых хранилищ "ключ-значение", работа с IndexedDB построена на асинхронных событиях и является значительно более сложной в реализации. В большинстве повседневных задач фронтенд-разработчику достаточно возможностей LocalStorage и Cookies.

### Итог

Мы изучили основные инструменты хранения данных на клиенте. Куки незаменимы для связи с сервером и авторизации, в то время как Web Storage API (LocalStorage и SessionStorage) предоставляет удобный и емкий механизм для хранения пользовательских настроек и состояния интерфейса прямо в браузере. Выбор инструмента зависит от того, как долго должны жить данные и нужно ли их передавать серверу при каждом запросе.

### Задача на закрепление

Создайте простую систему сохранения текста в поле ввода.

1. Создайте HTML-страницу с элементом `textarea`.
2. Реализуйте функционал, при котором каждое введенное слово автоматически сохраняется в `localStorage` (используйте событие `input`).
3. При перезагрузке страницы текст в `textarea` должен восстанавливаться из хранилища.
4. Добавьте кнопку "Очистить черновик", которая удаляет данные из `localStorage` и очищает поле ввода.

**Критерии успешного выполнения:**

* Текст не пропадает после обновления страницы.
* Данные в хранилище обновляются корректно при вводе.
* Кнопка очистки полностью удаляет следы черновика как из интерфейса, так и из памяти браузера.
