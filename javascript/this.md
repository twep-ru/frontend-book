# this

Понимание того, как работает `this`, необходимо каждому фронтенд-разработчику, так как это база для работы с объектами, создания повторно используемого кода и понимания того, как функционируют современные фреймворки и библиотеки. `this` позволяет функциям получать доступ к данным того объекта, в контексте которого они были вызваны, делая код более гибким и динамичным.

### Понятие глобального контекста

Когда мы используем `this` вне каких-либо функций или объектов, оно ссылается на глобальный объект. В среде браузера этим объектом является `window`. Он содержит в себе множество встроенных методов, таких как `alert()` или `fetch()`, а также информацию о текущем окне браузера. Если вы просто напишете `console.log(this)` в глобальной области видимости, вы увидите структуру этого огромного объекта. Однако в реальной разработке `this` чаще всего используется внутри функций, которые привязаны к конкретным объектам.

### Ключевое слово this в методах объекта

Наиболее частое применение `this` находит в методах — функциях, которые являются свойствами объекта. В этом случае `this` ссылается на «объект перед точкой», то есть на тот самый объект, у которого вызывается метод. Это позволяет методу манипулировать данными внутри своего же объекта.

```javascript
const user = {
  name: 'Александр',
  age: 25,
  logInfo() {
    console.log(`Имя: ${this.name}, Возраст: ${this.age}`);
  }
};

user.logInfo(); // Выведет: Имя: Александр, Возраст: 25
```

В приведенном примере `this.name` эквивалентно обращению к `user.name`. Важно понимать, что значение `this` вычисляется в момент вызова функции. Если одну и ту же функцию назначить методам разных объектов, то в каждом из них `this` будет указывать на свой объект.

### Поведение со стрелочными функциями

Особое внимание стоит уделить стрелочным функциям, так как они ведут себя иначе. В отличие от обычных функций, стрелочные функции не имеют своего собственного `this`. Они «заимствуют» его из внешней области видимости, где были созданы. Если вы попытаетесь использовать стрелочную функцию в качестве метода объекта для обращения к его свойствам, то получите `undefined` (или глобальный объект), так как контекст объекта для них не создается.

```javascript
const user = {
  name: 'Иван',
  sayHi: () => {
    console.log(this.name); // Ошибка или undefined, контекст потерян
  }
};
```

Для корректной работы с данными объекта рекомендуется использовать либо классический синтаксис `function`, либо современную сокращенную запись метода, которая сохраняет правильный контекст.

### Цепочки вызовов (Chaining)

Иногда в разработке встречается паттерн, называемый «чейнингом» (от англ. chain — цепь). Это способ вызывать несколько методов объекта один за другим через точку. Чтобы это стало возможным, каждый метод объекта должен в конце своего выполнения возвращать само ключевое слово `this`. Это позволяет среде выполнения «увидеть» объект снова и вызвать у него следующий метод.

```javascript
const ladder = {
  step: 0,
  up() {
    this.step++;
    return this;
  },
  down() {
    this.step--;
    return this;
  },
  showStep() {
    console.log(this.step);
    return this;
  }
};

// Благодаря return this возможна такая запись:
ladder.up().up().down().showStep(); // Выведет 1
```

Такой подход делает код более лаконичным и читаемым, и он часто используется в популярных библиотеках, например, для анимаций или обработки данных.

### Итог

Ключевое слово `this` в JavaScript служит ссылкой на контекст выполнения функции, который определяется в момент её вызова. В методах объектов оно указывает на сам объект, позволяя работать с его внутренним состоянием, в то время как стрелочные функции игнорируют контекст вызова, сохраняя связь с внешним окружением. Освоение работы с `this` и понимание механизмов возврата контекста для создания цепочек вызовов открывают путь к написанию чистого и эффективного объектно-ориентированного кода.

### Задача на закрепление

Создайте объект `accumulator`, который будет имитировать работу накопителя значений. Объект должен иметь свойство `value`, в котором хранится текущая сумма (изначально 0). Реализуйте следующие методы:

1. `add(number)` — прибавляет переданное число к текущему значению `value`.
2. `subtract(number)` — вычитает число из `value`.
3. `showValue()` — выводит текущее значение в консоль.
4. Модифицируйте методы так, чтобы можно было вызывать их по цепочке.

**Критерии успешного выполнения:**

* Создан объект с начальным свойством `value: 0`.
* Методы `add` и `subtract` корректно изменяют `value` через `this`.
* Реализована возможность вызова в формате: `accumulator.add(10).subtract(3).showValue();`.
* В консоли отображается итоговый результат (в данном примере — 7).
