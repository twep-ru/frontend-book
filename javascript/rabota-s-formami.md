# Работа с формами

Через них мы получаем регистрационные данные, поисковые запросы, комментарии и настройки профиля. Для фронтенд-разработчика критически важно уметь не просто находить элементы форм в DOM-дереве, но и грамотно манипулировать их состоянием, читать введённые значения и связывать разрозненные элементы интерфейса в единую логическую структуру.

### Доступ к формам и их элементам

Существует несколько способов получить доступ к форме на странице. Помимо стандартных методов `querySelector`, JavaScript предоставляет специальное свойство `document.forms`, которое возвращает коллекцию всех форм в документе. К конкретной форме можно обратиться по индексу или, что более профессионально, по значению атрибута `id` или `name`, если они указаны в разметке.

Внутри каждой формы есть свойство `elements`. Оно содержит живую HTML-коллекцию всех интерактивных элементов: инпутов, текстовых областей, кнопок и селектов. Преимущество `elements` перед обычным поиском через `querySelectorAll` заключается в том, что коллекция всегда остается актуальной (живой), даже если элементы добавляются или удаляются динамически. Кроме того, к любому полю ввода внутри формы можно обратиться напрямую через её имя (атрибут `name`), что делает код лаконичным: `myForm.login` вместо длинных путей поиска.

### Связывание элементов через атрибут form

Иногда дизайн приложения требует, чтобы кнопка или поле ввода находились визуально вне тега `<form>`. В таких случаях на помощь приходит атрибут `form`. Если мы укажем у инпута атрибут `form="id-вашей-формы"`, этот элемент логически привяжется к указанной форме, даже если физически находится в другой части страницы. В JavaScript такой "внешний" элемент появится в коллекции `elements` данной формы, а его собственное свойство `.form` будет ссылаться на соответствующий DOM-элемент формы.

### Чтение и изменение значений: Текст и Радиокнопки

Для работы с текстовыми полями (`input type="text"`) и областями `textarea` используется свойство `value`. Оно позволяет как прочитать текущий текст, введенный пользователем, так и программно установить новое значение. Важно помнить, что для `textarea` всегда следует использовать именно `.value`, а не `innerText` или `textContent`, чтобы гарантированно получать актуальные данные.

Работа с радиокнопками (`type="radio"`) имеет свои нюансы. Поскольку группа радиокнопок объединена одним именем `name`, при обращении к ним через `form.elements.name` мы получаем не один элемент, а коллекцию `RadioNodeList`. У этой коллекции есть удобное свойство `value`, которое возвращает значение (`value`) именно той кнопки, которая выбрана в данный момент. Чтобы это работало корректно, у каждой радиокнопки в HTML обязательно должен быть прописан атрибут `value`.

### Чекбоксы и выпадающие списки

В отличие от текстовых полей, состояние чекбокса (`type="checkbox"`) определяется свойством `checked`, которое возвращает булево значение: `true`, если галочка стоит, и `false`, если нет. Обращение к `value` у чекбокса без выбранного состояния часто возвращает неинформативную строку "on".

Для элементов `select` свойство `value` также является основным способом получения выбранной опции. Однако, если селект поддерживает множественный выбор (атрибут `multiple`), свойство `value` вернет только первое выбранное значение. В таких случаях необходимо использовать свойство `selectedOptions`, которое содержит коллекцию всех выбранных элементов `<option>`, и перебирать их (например, с помощью метода `map`, предварительно превратив коллекцию в массив) для получения полного списка данных.

### Итог

Мы изучили механизмы взаимодействия с формами, которые позволяют эффективно управлять пользовательским вводом. Основные инструменты разработчика здесь — это коллекции `document.forms` и `form.elements`, которые обеспечивают быстрый доступ к полям по их именам. Понимание разницы между свойствами `value` и `checked`, а также умение использовать атрибут `form` для связки элементов вне контейнера, составляют базу для создания сложных и удобных интерфейсов.

### Задача на закрепление

Создайте форму регистрации пользователя и напишите скрипт для управления её данными.

1. Создайте HTML-форму с `id="regForm"`. Внутри должны быть:
   * Текстовое поле для логина (`name="login"`).
   * Группа из двух радиокнопок для выбора роли: "User" и "Admin" (`name="role"`).
   * Чекбокс "Согласен с условиями" (`name="policy"`).
2. Вне тега формы создайте элемент `<select name="city">` с несколькими городами и привяжите его к форме с помощью атрибута `form`.
3. Напишите функцию в JS, которая:
   * Выводит в консоль текущее значение логина.
   * Проверяет, выбрана ли роль "Admin", и если да — устанавливает значение чекбокса "Согласен с условиями" в `true`.
   * Изменяет выбранный город в селекте на второй по списку программным способом.
4. Проверьте работоспособность, обращаясь к элементам только через свойство `.elements` вашей формы.

**Критерии успешного выполнения:**

* Доступ к полям осуществляется через `document.forms.regForm.elements`.
* Внешний селект корректно отображается в коллекции элементов формы.
* Скрипт успешно меняет состояние чекбокса и значение селекта.
