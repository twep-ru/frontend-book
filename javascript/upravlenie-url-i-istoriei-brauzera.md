# Управление URL и историей браузера

Понимание того, как программно изменять URL без перезагрузки документа и управлять тем, что происходит при взаимодействии с браузерными кнопками навигации, является критически важным навыком для фронтенд-разработчика. Это позволяет сохранять текущее состояние интерфейса прямо в ссылке, чтобы пользователи могли делиться ею или сохранять в закладки, не теряя при этом выбранных фильтров, открытых модальных окон или текущей вкладки.

### Взаимодействие с текущим адресом через Location API

Глобальный объект `window` предоставляет доступ к свойству `location`, которое содержит исчерпывающую информацию о текущем URL-адресе и позволяет им управлять. При обращении к `window.location` мы получаем доступ к объекту с различными частями адреса. Свойство `href` содержит полный URL, `protocol` указывает на используемый протокол (http или https), `host` включает в себя имя домена и порт, а `pathname` отражает путь к странице после доменного имени.

Особое внимание стоит уделить свойствам `search` и `hash`. Свойство `search` содержит строку GET-параметров, которая начинается с вопросительного знака. Это та самая часть адреса, где передаются данные различных фильтров, сортировок или поисковых запросов. Свойство `hash` хранит часть адреса, начинающуюся с символа решетки, которая чаще всего используется для якорных ссылок, ведущих к определенному блоку на текущей странице.

Помимо чтения информации, Location API предоставляет методы для принудительного изменения адреса. Метод `reload()` позволяет перезагрузить текущую страницу. Методы `assign()` и `replace()` используются для программного перехода на новый URL. Ключевое различие между ними заключается в том, что `assign()` сохраняет текущую страницу в истории браузера, позволяя пользователю вернуться назад, в то время как `replace()` перезаписывает текущую запись в истории сессии, делая возврат на предыдущую страницу через кнопку «Назад» невозможным. Присвоение нового значения напрямую свойству `href` по умолчанию сработает аналогично вызову `assign()`.

```javascript
// Перезагрузка текущей страницы
window.location.reload();

// Переход на новую страницу с сохранением истории
window.location.assign('https://example.com/catalog');

// Переход без сохранения текущей страницы в истории
window.location.replace('https://example.com/about');

// Альтернативный и популярный способ перехода
window.location.href = 'https://example.com/contact';
```

### Управление историей сессии через History API

Объект `window.history` предоставляет функционал для взаимодействия с историей переходов пользователя в рамках текущей вкладки браузера. Свойство `length` этого объекта показывает общее количество записей в истории текущей сессии.

Для навигации по уже существующей истории предусмотрены методы `back()` и `forward()`, которые программно имитируют нажатие соответствующих браузерных стрелок. Если необходимо переместиться на несколько шагов сразу, используется метод `go()`, принимающий в качестве аргумента число, указывающее направление и количество шагов (например, `-2` для перехода на две страницы назад).

Наибольшую ценность для разработчиков представляют методы `pushState()` и `replaceState()`. Именно они позволяют изменять URL-адрес в адресной строке без фактической перезагрузки страницы. Метод `pushState()` добавляет новую запись в историю сессии. Он принимает три аргумента: объект состояния (любые данные, которые вы хотите привязать к новой записи), строку (в современных браузерах этот параметр игнорируется, поэтому туда передают пустую строку) и сам новый URL-адрес. Метод `replaceState()` принимает идентичные аргументы, но вместо создания новой записи он бесшумно модифицирует последнюю. Это невероятно полезно при реализации фильтрации товаров: когда пользователь выбирает новые параметры, мы обновляем URL через `replaceState()`, чтобы ссылка всегда была актуальной, не засоряя при этом историю браузера сотнями промежуточных кликов по чекбоксам.

```javascript
// Программный переход на одну страницу назад
window.history.back();

// Программный переход на два шага вперед по истории
window.history.go(2);

// Добавление новой записи в историю без перезагрузки документа
window.history.pushState({ page: 'catalog' }, '', '/catalog?sort=price');

// Обновление текущей записи (идеально для синхронизации фильтров)
window.history.replaceState({ filter: 'active' }, '', '/catalog?color=red');
```

### Удобная работа с параметрами с помощью URLSearchParams

Работать с GET-параметрами напрямую через строку `window.location.search` бывает неудобно, так как требуется вручную разбивать текст по символам амперсанда и знака равенства. Для элегантного решения этой задачи существует встроенный класс `URLSearchParams`.

При создании экземпляра этого класса мы передаем ему строку с параметрами. В ответ мы получаем специальную структуру данных, предоставляющую удобные методы `get`, `set`, `append`, `has` и `delete` для манипуляции параметрами. С помощью цикла `for...of` можно легко перебрать все доступные ключи и их значения. Этот класс также отлично работает в связке с объектом `FormData`: передав собранные данные формы в конструктор `URLSearchParams`, мы можем моментально сгенерировать корректную строку GET-параметров с помощью метода `toString()`, что кардинально упрощает процесс переноса состояния интерфейса в URL-адрес.

```javascript
// Парсинг существующих параметров из текущего URL
const params = new URLSearchParams(window.location.search);

for (const [key, value] of params) {
  console.log(`Ключ: ${key}, Значение: ${value}`);
}

// Генерация строки параметров на основе данных формы
const formData = new FormData(document.querySelector('.filter-form'));
const newParams = new URLSearchParams(formData);
const queryString = newParams.toString();

// Синхронизация нового URL с текущим состоянием формы
window.history.replaceState({}, '', `${window.location.pathname}?${queryString}`);
```

### Итог

Мы изучили мощные инструменты браузерного окружения, позволяющие полностью контролировать адресную строку и процесс навигации. Location API дает возможность детально анализировать текущий URL и инициировать переходы с классической перезагрузкой документа. History API открывает двери к созданию современных интерфейсов, позволяя манипулировать историей сессии и менять адрес "на лету", что является абсолютной основой маршрутизации в любом SPA-приложении. Класс URLSearchParams избавляет от рутинной обработки строк при парсинге и формировании GET-параметров, делая код чище и надежнее. В совокупности эти технологии позволяют связывать внутреннее состояние веб-приложения с ссылкой, обеспечивая превосходный и бесшовный пользовательский опыт.

### Задача на закрепление

Создайте простую HTML-страницу с кнопкой и пустым элементом `div` для вывода текста. Напишите JavaScript-код, который реализует программное изменение URL и немедленно реагирует на него. При клике на кнопку необходимо с помощью метода `pushState` добавить в адресную строку GET-параметр `modal` со значением `login` (например, ваш URL должен стать `?modal=login`), не вызывая перезагрузку страницы. Сразу после изменения URL, ваш код должен прочитать новые GET-параметры с использованием класса `URLSearchParams` и вывести значение параметра `modal` внутри созданного пустого `div`.

**Критерии успешного выполнения:**

* При нажатии на кнопку URL-адрес в строке браузера мгновенно обновляется без визуального моргания или перезагрузки страницы.
* В истории браузера появляется новая запись, что позволяет нажать стандартную кнопку браузера «Назад» для возврата к изначальному URL.
* На странице корректно отображается слово "login", а логика чтения этого параметра реализована строго через методы класса `URLSearchParams`.
