# Объекты, часть 2

Работа с объектами является фундаментом современной фронтенд-разработки. Понимание того, как объекты хранятся в памяти, как их правильно копировать и извлекать из них данные, критически важно для написания предсказуемого и чистого кода. Без этих знаний разработчик неизбежно столкнется с трудноуловимыми ошибками, когда изменение одной переменной внезапно «ломает» данные в другой части приложения.

### **Ссылочный тип данных и особенности сравнения**

В JavaScript объекты относятся к ссылочным типам данных. В отличие от примитивов (строк, чисел), которые копируются по значению, переменная объекта хранит не сами данные, а адрес области памяти, где этот объект расположен.

Это приводит к тому, что два внешне идентичных объекта при сравнении через `==` или `===` всегда дадут `false`. Для движка JavaScript это две разные «коробки» в памяти, даже если их содержимое совпадает. Сравнение вернет `true` только в том случае, если обе переменные ссылаются на один и тот же участок памяти.

```javascript
const user1 = { name: 'Alex' };
const user2 = { name: 'Alex' };

console.log(user1 === user2); // false

const user3 = user1; 
console.log(user1 === user3); // true (ссылаются на один объект)
```

Чтобы сравнить содержимое двух разных объектов, необходимо выполнить их перебор. Простейший способ — сравнить количество ключей (используя `Object.keys()`) и затем в цикле сопоставить значения каждого свойства. Если внутри объекта находятся другие объекты, потребуется рекурсивный обход для «глубокого» сравнения.

### **Клонирование и объединение объектов**

Поскольку объекты копируются по ссылке, простое присваивание `obj2 = obj1` не создает новый объект. Если вы измените свойство в `obj2`, оно изменится и в `obj1`. Чтобы создать независимую копию, используются современные инструменты языка.

Наиболее популярным методом является Spread-оператор (три точки `...`). Он буквально «распаковывает» свойства старого объекта в новый. Также существует метод `Object.assign()`, который выполняет аналогичную задачу, копируя свойства из одного или нескольких исходных объектов в целевой.

```javascript
const defaults = { theme: 'light', fontSize: 16 };
const userConfig = { fontSize: 18, showSidebar: true };

// Объединение объектов: свойства из userConfig перезапишут совпадения из defaults
const finalConfig = { ...defaults, ...userConfig }; 
// Результат: { theme: 'light', fontSize: 18, showSidebar: true }
```

Важно помнить, что и Spread, и `Object.assign()` выполняют поверхностное копирование. Если внутри объекта есть вложенные объекты, они все равно будут скопированы по ссылке. Для полного отделения объектов друг от друга на всех уровнях вложенности требуется глубокое клонирование.

### **Опциональная цепочка (Optional Chaining)**

Фронтенд-разработчики часто работают с данными от сервера, структура которых может меняться или быть неполной. Попытка обратиться к свойству несуществующего объекта (например, `user.address.city`, если `address` равен `undefined`) приведет к критической ошибке и остановке всего приложения.

Оператор опциональной цепочки `?.` позволяет безопасно обращаться к глубоко вложенным свойствам. Если какая-то часть цепочки отсутствует, выражение просто вернет `undefined`, не вызывая ошибку.

```javascript
const user = { name: 'Ivan' };

// Вместо ошибки получим undefined
console.log(user.address?.city); 

// Часто комбинируется с оператором нулевого слияния (??) для задания значения по умолчанию
const city = user.address?.city ?? 'Город не указан';
```

### **Деструктуризация и остаточные параметры**

Деструктуризация — это синтаксис, позволяющий «разобрать» объект на отдельные переменные. Это делает код лаконичнее, избавляя от необходимости многократно писать `user.name`, `user.age` и так далее. Мы можем задавать переменным новые имена или устанавливать значения по умолчанию прямо в процессе извлечения.

```javascript
const profile = { username: 'dev_user', age: 25 };

// Извлекаем username в переменную name, задаем значение по умолчанию для isAdmin
const { username: name, isAdmin = false } = profile;

console.log(name); // 'dev_user'
console.log(isAdmin); // false
```

Этот синтаксис особенно полезен в параметрах функций. Вместо того чтобы передавать пять аргументов в строгом порядке, можно передать один объект и деструктурировать его прямо в заголовке функции. Это повышает читаемость и защищает от ошибок, связанных с очередностью аргументов.

Если же нам нужно забрать пару свойств, а остальные сохранить в отдельный объект, используется Rest-оператор. Он собирает все оставшиеся поля в новую структуру.

```javascript
const settings = { color: 'red', width: 100, height: 200, opacity: 0.5 };
const { color, ...dimensions } = settings;

console.log(dimensions); // { width: 100, height: 200, opacity: 0.5 }
```

### **Итог**

Глубокое понимание работы с объектами позволяет разработчику эффективно управлять состоянием приложения. Ссылочная природа объектов требует осознанного подхода к их сравнению и копированию. Использование Spread-оператора, деструктуризации и опциональной цепочки значительно сокращает объем шаблонного кода, делает его более безопасным при работе с внешними данными и упрощает поддержку сложных систем.

### Задача на закрепление

Представьте, что вы разрабатываете систему управления заказами. У вас есть объект заказа, полученный от сервера, который может содержать вложенную информацию о клиенте и доставке.

**Задание:**

1. Создайте объект `order` со свойствами `id`, `status` и вложенным объектом `customer` (поля `fullName` и `contacts`). Поле `contacts` может отсутствовать.
2. Используя деструктуризацию, извлеките `id` заказа и `fullName` клиента (переименуйте его в `clientName`).
3. Используя опциональную цепочку и оператор нулевого слияния, создайте переменную `email`, которая будет содержать `customer.contacts.email`, либо строку "Email не указан", если данные отсутствуют.
4. Создайте копию объекта `order`, изменив в ней статус на "completed", не меняя оригинальный объект.

**Критерии успешного выполнения:**

* Оригинальный объект `order` не должен измениться после создания копии.
* Код не должен выбрасывать ошибок, даже если в объекте `order` отсутствует свойство `customer`.
* Использованы синтаксические конструкции: деструктуризация с переименованием, Spread-оператор, Optional Chaining и Nullish Coalescing.
