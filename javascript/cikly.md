# Циклы

Например, это может быть вывод списка ссылок в меню навигации или повторение операции до тех пор, пока не выполнится заданное условие. Во всем этом помогают циклы — специальные конструкции языка, которые позволяют автоматизировать рутинные задачи и избежать дублирования кода. Понимание работы циклов критически важно, так как они лежат в основе алгоритмики и манипуляций с данными в JavaScript.

### Цикл while

Первый и самый базовый цикл — `while`. Он выполняет блок кода до тех пор, пока условие, описанное в круглых скобках, остается истинным (соответствует логическому значению `true`). Как только условие становится ложным (`false`), выполнение цикла прекращается.

```javascript
let count = 0;
while (count < 10) {
  console.log(count);
  count++;
}
```

В этом примере создается счетчик `count` с начальным значением ноль. В условии `while` проверяется, что пока `count` меньше десяти, будет выполняться код внутри фигурных скобок: вывод текущего значения в консоль и последующее увеличение его на единицу с помощью оператора инкремента. Если начальное значение переменной сходу сделать равным ста, условие сразу окажется ложным, и тело цикла не выполнится ни разу.

Стоит быть предельно осторожным при работе с циклом `while`. Если в теле цикла забыть увеличить счетчик, условие никогда не станет ложным, что приведет к бесконечному выполнению кода. В браузере такая ситуация чревата серьезной утечкой памяти: вкладка начнет накапливать используемые сущности, потребляя все доступные ресурсы оперативной памяти устройства, пока полностью не зависнет. Именно из-за этого риска `while` используется в реальных проектах относительно редко.

### Цикл do...while

Иногда возникают ситуации, когда необходимо выполнить код из тела цикла как минимум один раз, даже если условие изначально является ложным. Для этих целей применяется немного другая вариация — `do...while`.

```javascript
let count = 0;
do {
  console.log(count);
  count++;
} while (count < 10);
```

Эта конструкция сначала выполняет блок кода, обернутый в фигурные скобки после ключевого слова `do`, а уже затем проверяет условие в конце конструкции `while`. Если условие оказывается истинным, цикл повторяется, и так происходит до тех пор, пока проверка не вернет `false`.

### Цикл for и область видимости

Гораздо чаще на практике используется цикл `for`. Он крайне удобен тем, что позволяет в одних круглых скобках компактно объединить объявление счетчика, условие выполнения и код, выполняющийся в конце каждой итерации.

```javascript
for (let i = 0; i < 4; i++) {
  console.log(i);
}
```

Инструкция `for` состоит из трех частей, разделенных точкой с запятой. Первая часть — это начало цикла, которое выполняется ровно один раз при запуске. Здесь обычно объявляют переменную-счетчик, традиционно называемую одной буквой `i`, `j` или `k`. Вторая часть содержит условие, которое проверяется перед каждым шагом: если оно истинно, цикл продолжается, если ложно — останавливается. Третья часть представляет собой код, выполняющийся в конце каждой итерации (чаще всего это увеличение счетчика).

При работе с `for` важно понимать концепцию области видимости переменных. Если переменная счетчика объявлена внутри круглых скобок цикла (или внутри его тела), получить к ней доступ за пределами фигурных скобок уже не получится. При попытке вывести такую переменную в консоль после завершения цикла, JavaScript выдаст ошибку о том, что переменная не объявлена.

### Прерывание и продолжение цикла

Работу любого цикла можно гибко контролировать изнутри с помощью специальных ключевых слов. Если выполнение цикла больше не требуется и его необходимо досрочно завершить (например, при поиске конкретного значения), используется `break`.

```javascript
for (let i = 1; i <= 10; i++) {
  if (i === 5) {
    break; 
  }
  console.log(i);
}
```

Эта механика позволяет отлично оптимизировать код, так как прерывает работу цикла сразу после срабатывания нужного условия, не тратя ресурсы на оставшиеся итерации.

Если же нужно пропустить только текущую итерацию и мгновенно перейти к следующему шагу, не выполняя оставшийся код в теле цикла, применяется `continue`.

```javascript
for (let i = 1; i <= 10; i++) {
  if (i % 2 === 0) {
    continue; 
  }
  console.log(i); 
}
```

В данном примере проверка остатка от деления позволяет отфильтровать все четные числа, прерывая шаг для них с помощью `continue`. В результате в консоль попадут исключительно нечетные значения.

### Итог

Циклы являются фундаментальным инструментом разработки, позволяющим многократно выполнять блоки кода без дублирования. Конструкции `while` и `do...while` отлично подходят для повторений с плавающим количеством итераций, зависящим от условия, в то время как `for` предлагает строгую и компактную структуру для работы со счетчиками. Понимание области видимости защищает код от непредвиденных ошибок, а умение досрочно завершать работу цикла инструкцией `break` или пропускать шаги с помощью `continue` делает программы более оптимизированными и гибкими.

### Задача на закрепление

Напишите программу, которая выводит в консоль числа от 1 до 15 включительно, используя цикл `for`.

Внутри тела цикла добавьте следующие условия:

1. Если текущее число кратно 3 (остаток от деления на 3 равен нулю), пропустите вывод этого числа в консоль и перейдите к следующей итерации.
2. Если текущее число достигло 12, выведите его в консоль и полностью остановите выполнение цикла, чтобы числа 13, 14 и 15 не обрабатывались.

**Критерии успешного выполнения:**

* Использован цикл `for` с корректно заданными начальным значением, условием и шагом.
* В консоли отсутствуют числа, кратные 3 (выводятся числа: 1, 2, 4, 5, 7, 8, 10, 11).
* Цикл досрочно прерывается на числе 12 (само число 12 должно быть выведено), и после него в консоль ничего не выводится.
* В коде правильно применены инструкции `continue` и `break`.
